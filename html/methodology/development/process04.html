개요

리팩토링 과정을 통해 설계 및 코드 개선을 실시 함

  • 설계와 구현은 동일한 시간 축에서 진행되며 상호 피드백 구조를 가짐
  • 리팩토링 과정을 통해서 설계 및 코드 개선을 시도함

활동(Activity)명

활동 내용

관련 산출물

5.1 서버 컴포넌트 구현

아키텍처 설계와 컴포넌트 설계를 참조하여 구현함. 반드시 테스트케이스 개발과 병행

서버 컴포넌트

5.2 UI 컴포넌트 구현

UI와 프리젠테이션 영역을 포함하는 일명 웹컴포넌트를 구현함. 단위테스트 병행

UI 컴포넌트

5.3 코드 리뷰

주요 컴포넌트에 대한 코드 리뷰를 수행하고 리뷰결과 피드백을 반영함

코드 리뷰 결과서

5.4 리팩토링 및 개선

구현과정에서 파악된 설계 개선 사항에 대해 컴포넌트 명세, 설계, 코드 등에 대해 작업

5.4.1 컴포넌트 명세 개선

구현과정에서 발견한 인터페이스 추가, 인터페이스 정보 변경 등을 명세에 반영함

[개선된] 컴포넌트 명세

5.4.2 컴포넌트 리팩토링

식별된 컴포넌트 내부 구조상의 문제에 대해 수시로 소규모 개선을 실시함

[개선된] 컴포넌트

5.5 컴포넌트 통합

의존관계에 있는 컴포넌트를 통합하여 작업을 수행함

[검증된] 컴포넌트

5.6 단위 테스트

각 컴포넌트 인터페이스 단위로 단위 테스트를 수행하여 코드 커버리지 비율 준수

단위 테스트 코드

5.7 데이터 이행

AS-IS 데이터가 있을 경우 신규 시스템에 맞추어 이행작업을 수행함

데이터 이행 계획서

5.8 테스트 준비

[개선된] 테스트 계획서

5.8.1 테스트 케이스 개선

기 작성된 테스트 케이스와 관련 시나리오를 변경된 컴포넌트 명세에 맞추어 변경함

[개선된] 테스트 케이스

5.8.2 테스트 데이터 구축

테스트 케이스 수행에 필요한 테스트 데이터를 준비함

테스트 데이터

5.8.3 테스트 스크립트 작성

테스트 스크립트를 최종 확정된 컴포넌트 인터페이스를 기반으로 작성함

테스트 스크립트

5.8.4 테스트 환경 준비

요구되는 수준(운영/테스트)에 맞추어 테스트를 위한 H/W, S/W 환경을 준비함

테스트 계획서-환경부문

5.8.5 배치(deployment)

개발한 결과 시스템을 준비된 테스트 환경에 설치함

테스트 계획서-배치부문
