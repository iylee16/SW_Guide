단계별 활동, 결과산출물, 템플릿, 개발가이드

개발단계 활동 활동 내용 활동 결과산출물 산출물 양식 관련 기법 개발 도구
1. 요구사항 정의 1.1 문제기술서 작성 사용자의 관점에서 서술한 사용자의 일반적이고 포괄적인 요구사항을 명세함 문제기술서 요구사항명세서 요구사항개발 가이드(도출/분석/명세/검증)
1.2 용어 정의 목표시스템의 도메인 용어에 대한 정의를 나열함 용어사전
1.3 현행 시스템 분석 관련 현행 시스템이 제시될 결우 데이터, 비즈니스 로직, UI 등에 대한 분석을 수행함 현행 시스템 분석서
1.4 기능 요구사항 명세 Use Case를 이용한 기능요구사항 명세함 유스케이스 명세서
1.5 품질 요구사항 명세 비기능(=품질) 요구사항을 식별하고 명세함 품질 요구사항 명세서
1.6 개발환경 준비 개발에 필요한 환경(IDE, 지원도구, 주요 컴포넌트 등)을 준비함 통합개발환경 정의서
2. 분석 2.1 비즈니스 로직 분석 다양한 관점의 활동을 통해서 요구사항을 분석하고 설계를 위한 기초를 다짐 도메인 객체 모델 요구사항명세서 요구사항개발 가이드(도출/분석/명세/검증) UX/UI표준 가이드 요구사항 관리 - OSRMT
2.2 UI 분석 고객이 가장 쉽게 받아들이는 UI를 통해 요구사항을 UI 분석 모델로 발전시킴 UI 분석 명세서
2.3 테스트 계획서 작성 분석 결과 파악한 시스템의 범위, 특성, 등을 기반으로 테스트 계획서를 작성함 테스트 계획서 테스트 계획서 테스트 전략 가이드
2.4 테스트 케이스 작성 분석 자료를 이용하여 시나리오를 포함하는 테스트 케이스 작성 테스트 케이스 테스트케이스명세서 테스트 설계 가이드
3.1 아키텍처 문서 작성 아키텍처 방향, 뷰, 결정내용과 근거, 주요 기술구조 등을 포함하여 뷰를 중심으로 기술함 아키텍처 정의서 아키텍처 설계서 아키텍처 설계 가이드 설계/모델링 - StarUML
3.2 아키텍처 설계 UML을 이용하여 아키텍처를 뷰 단위로 모델링함. 아키텍처 요소(프레임워크 등)에 대한 모델링을 포함함 아키텍처 프로토타입
3.3 아키텍처 프로토타입 개발 아키텍처 프로토타입을 참조용[또는 검증용] 으로 개발함. 참조용으로 작성시 설계 모델구조와 소스코드를 정형화하여 모든 기능개발팀이 참조할 수 있도록 함 아키텍처 프로토타입
3.4 아키텍처 적용 가이드 작성 아키텍처를 준수하면서 설계를 수행할 수 있도록 다양한 가이드를 작성함. 간단한 프로젝트나 팀이 익숙한 기술을 사용하는 경우 아키텍처 문서로 대신함 설계가이드
3.4.3 프레임워크 사용 가이드 아키텍처 설계 과정에서 개발한 프레임워크(들)에 대한 사용 가이드 프레임워크 사용 가이드
3.5 인프라 아키텍처 설계 H/W와 네트워크 관련 기술 구조를 정의함 아키텍처 정의서에 포함
4. 설계 4.1 서버 컴포넌트 설계 아키텍처 방향, 뷰, 결정내용과 근거, 주요 기술구조 등을 포함하여 뷰를 중심으로 기술함 컴포넌트 설계모델 아키텍처 설계서

UX/UI 표준 가이드 		설계/모델링 - StarUML
4.2 데이터 설계 컴포넌트 설계를 기반으로 데이터 설계를 수행함 코드 정의서
4.3 UI 설계 UI 관련 설계를 수행함 UI 설계서
4.4 외부 인터페이스 설계 외부 시스템과의 인터페이스를 위한 전문, 인터페이스 기능지원 코드 작상 인부 인터페이스 설계서
4.5 설계 리뷰 설계 결과를 검토하고 피드백을 반영하여 설계를 개선함 설계 검토 결과서
5. 구현 5.1 서버 컴포넌트 구현 아키텍처 설계와 컴포넌트 설계를 참조하여 구현함. 반드시 테스트케이스 개발과 병행 서버 컴포넌트 소스코드

5.2 UI 컴포넌트 구현 UI와 프리젠테이션 영역을 포함하는 일명 웹컴포넌트를 구현함. 단위테스트 병행 UI 컴포넌트
5.3 코드 리뷰 주요 컴포넌트에 대한 코드 리뷰를 수행하고 리뷰결과 피드백을 반영함 코드 리뷰 결과서
5.4 리팩토링 및 개선 구현과정에서 파악된 설계 개선 사항에 대해 컴포넌트 명세, 설계, 코드 등에 대해 작업 [개선된] 컴포넌트
5.5 컴포넌트 통합 의존관계에 있는 컴포넌트를 통합하여 작업을 수행함 [검증된] 컴포넌트
5.6 단위 테스트 각 컴포넌트 인터페이스 단위로 단위 테스트를 수행하여 코드 커버리지 비율 준수 단위 테스트 코드
5.7 데이터 이행 AS-IS 데이터가 있을 경우 신규 시스템에 맞추어 이행작업을 수행함 데이터 이행 계획서
5.8 테스트 준비 테스트 케이스, 환경 등 실행 준비 [개선된] 테스트 계획서
6. 테스트 6.1 통합 테스트 여러 팀에서 개발한 다양한 컴포넌트를 통합 및 연계 관점에서 준비된 케이스와 시나리오를 이용하여 테스트 함 테스트 결과서 – 통합 부문 테스트케이스명세서
테스트 결과서 결함관리대장 		기능테스트 가이드 결함관리 가이드 결함유형분류 가이드 통합(기능)테스트 - Fitness
성능테스트 - Jmeter
6.2 시스템 테스트 시스템의 품질 속성 중에 하드웨어 또는 솔루션과 연관성이 있는 속성(성능, 가용성 등)에 대한 테스트 테스트 결과서 – 시스템 부문
6.3 인수 테스트 기능과 시스템, 운영 등의 관점에서 최종사용자가 허용할 만한 기준에 도달했는지 여부에 대한 테스트 테스트 결과서 – 인수 부문
6.4 테스트 결과 반영 테스트 결과서 – 변경 이력 테스트 결과서 – 변경 이력
7. 인도 7.1 지원문서 작성 최종사용자 및 운영자를 위한 다양한 문서를 작성함 사용자메뉴얼, 운영자메뉴얼, 릴리즈노트
7.2 사용자 교육 시스템 사용자 및 운영자에게 필요한 교육을 수행함 교육자료
