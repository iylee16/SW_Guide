개요

요구사항 정의 단계는 기능/비기능 요구사항을 정의하고 통합 개발 환경을 준비하는 단계로

  • 목표 시스템의 기능 요구, 비기능(=품질) 요구를 식별하고 명세한다.
  • 현행 시스템을 분석하고 도메인의 용어를 정의한다.
  • 시스템 구축(분석, 설계, 구현, 테스트) 환경을 준비한다.

활동(Activity)명

활동 내용

관련 산출물

1.1 이해관계자 요구사항 정의
(문제기술서 작성)

사용자의 관점에서 서술한 사용자의 일반적이고 포괄적인 요구사항을 명세함

문제기술서-이해관계자 요구사항 정의서

1.2 용어 정의

목표시스템의 도메인 용어에 대한 정의를 나열함

용어사전

1.3 현행 시스템 분석

관련 현행 시스템이 제시될 결우 데이터, 비즈니스 로직, UI 등에 대한 분석을 수행함

현행 시스템 분석서

1.4 기능 요구사항 명세

Use Case를 이용한 기능요구사항 명세함

유스케이스 명세서

유스케이스 모델

1.4.1 Use Case 식별

유스케이스를 식별하고 분류함

1.4.2 Use Case 명세서 작성

식별된 유스케이스를 표준 정보를 포함하여 명세함

1.5 품질 요구사항 명세

비기능(=품질) 요구사항을 식별하고 명세함

품질 요구사항 명세서

1.5.1 품질 속성 식별

품질 요구사항(성능, 가용성, 등)과 그 범위를 식별함

1.5.2 기능/품질 매핑

품질과 관련된 주요 기능 별 요구되는 품질속성과 수준 또는 범위를 기술함

1.6 개발환경 준비

개발에 필요한 환경(IDE, 지원도구, 주요 컴포넌트 등)을 준비함

통합개발환경 정의서

1.6.1 IDE 선정

개발에 사용할 표준 IDE 선정

1.6.2 지원도구 준비

IDE에 플러그-인하거나 별도로 사용할 다양한 지원도구 준비

1.6.3 IDE 운영 프로토타입

플러그-인이 통합된 IDE에서 간단한 프로젝트 개발 사례를 만들어서 공유함
