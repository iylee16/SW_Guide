프로세스(절차) - 아키텍처 설계

개요

아키텍처 뷰 단위 모델링 실시하며 아키텍처 요소(프레임워크 등)에 대한 모델링을 포함함

  • 상세 내역은 별도로 정의한 아키텍팅 프로세스를 참조함(이문서 버전은 아키텍처 프로토타입 기반 설계에 사용하는 방법임)
  • 아키텍팅 프로세스 역시 조정이 가능하도록 정의함

활동(Activity)명

활동 내용

관련 산출물

3.1 아키텍처 문서 작성

아키텍처 방향, 뷰, 결정내용과 근거, 주요 기술구조 등을 포함하여 뷰를 중심으로 기술함

아키텍처 정의서

아키텍처 프로토타입

--------------------

설계가이드

코딩규약

프레임워크 사용 가이드

3.2 아키텍처 설계

UML을 이용하여 아키텍처를 뷰 단위로 모델링함. 아키텍처 요소(프레임워크 등)에 대한 모델링을 포함함

3.3 아키텍처 프로토타입 개발

아키텍처 프로토타입을 참조용[또는 검증용] 으로 개발함. 참조용으로 작성시 설계 모델구조와 소스코드를 정형화하여 모든 기능개발팀이 참조할 수 있도록 함

3.4 아키텍처 적용 가이드 작성

아키텍처를 준수하면서 설계를 수행할 수 있도록 다양한 가이드를 작성함. 간단한 프로젝트나 팀이 익숙한 기술을 사용하는 경우 아키텍처 문서로 대신함

3.4.1 설계 가이드 작성

아키텍처 안에서 컴포넌트를 어떻게 설계 할 것인가에 대한 설명을 포함함

3.4.2 코딩 규약 작성

구현 언어의 코딩 규약을 작성함. 조직의 표준규약집이 있을 경우 대체함

3.4.3 프레임워크 사용 가이드

아키텍처 설계 과정에서 개발한 프레임워크(들)에 대한 사용 가이드

3.5 인프라 아키텍처 설계

H/W와 네트워크 관련 기술 구조를 정의함

아키텍처 정의서에 포함

활동 정의

활동(Activity)명 3.1 아키텍처 문서 작성
목적

아키텍처 요구사항, 접근방법, 아키텍처 설계와 원칙 등을 문서로 작성함

  • 모든 이해관계자의 관심사를 반영하여 아키텍처를 설계함
  • 설계 내용을 공유함
  • 준수해야 할 가이드라인으로서의 아키텍처 설계를 배포함
절차
  1. 준비: 아키텍처 요구사항 파악 → 아키텍처 자산 확인 → 설계 접근방법 결정
  2. 설계: 아키텍처 모델링 → 아키텍처 프로토타입 → 아키텍처 문서화

활동(Activity)명 3.2 아키텍처 설계
목적

제약조건 하에서 시스템의 품질요구를 만족하기 위하여 아키텍처 설계를 수행함. 아키텍처 설계는 방향설정, 주요 의사결정, 아키텍처 모델링, 설계 원칙 정의 등의 작업을 포함

  • 시스템의 구조 설계를 위한 원칙을 정의하고 그원칙에 따라 구조를 설계함
  • 가능한 여러 가지 문제해결 방법 중에 자원 효율적인 방안을 선택함
절차
  1. 시스템을 여러 영역으로 구분하고 설계 주제를 정의함
  2. 적용할 뷰를 정의
  3. 뷰를 중심으로 또는 설계 주제를 중심으로 구조를 설계함
  4. 구조 설계를 하면서 지속적으로 원칙을 추출하여 계층구조를 가지도록 정의함

활동(Activity)명 3.3 아키텍처 프로토타입 개발
목적

아키텍처 설계 결과를 검증하기 위해 실행 가능한 프로토타입 시스템을 개발함

  • 아키텍처 설계가 목표 시스템이 요구하는 품질목표를 달성했는지 검증함
  • 시스템 개발 팀이 개발에 참조할 시스템을 개발함
  • 아키텍처 설계에서 다루었던 이슈를 모두 포함하는 구조의 가능성 여부 검증
절차
  1. 아키텍처 관점에서 의미가 있는 주요 유스케이스 2,3개 식별
  2. 분석, 설계, 구현 과정을 거쳐 시스템 개발
  3. 품질검증에 초점을 두고 테스트 수행

활동(Activity)명 3.4 아키텍처 적용 가이드 작성
목적

아키텍처 설계 결과 적용 시 필요한 각종 가이드를 작성함

  • 아키텍처 설계도 중요하지만 준수가 매우 중요함 따라서 준수를 위한 도움을 제공하여야 함
  • 다양한 가이드를 제공함으로써 아키텍처를 준수하며 기능 설계를 할 수 있도록 지원함
절차
  1. 설계 가이드 작성
  2. 코딩 규약 작성
  3. 프레임워크 사용 가이드
역할자 소프트웨어 아키텍트 또는 아키텍처팀
산출물
  • 설계 가이드, 코딩 규약, 프레임워크 사용 가이드
  • 가이드 작성 범위는 조직의 표준 가이드 유무나 프로젝트의 성격에 따라 매우 다양함

활동(Activity)명 3.5 인프라 아키텍처 설계
목적

대규모 프로젝트에서 인프라 아키텍처 부분은 운영팀의 역할이지만 소규모 프로젝트에서는 소프트웨어 아키텍처 설계에 포함하여 수행함

  • 하드웨어 스팩, 구성, 네트워크 구성 등에 대한 설계를 수행함
  • 개발되거나 또는 구매한 소프트웨어 어느 하드웨어에 배포(deployment)되는 지를 결정함
  • 클러스터링 등을 포함한 플랫폼 SW의 구성을 결정함
절차
  1. 품질 요구사항 검토(범위성, 수행성능, 가용성 등)
  2. 가용 하드웨어 자산과 네트워크 및 보안 인프라 검토
  3. 인프라 아키텍처 설계
