개요

아키텍처 설계를 기반으로 상세 설계를 수행

  • 아키텍처 설계를 기반으로 서버/UI 컴포넌트 세부 설계를 수행함
  • 데이터 설계는 컴포넌트 설계를 기반으로 하여야 함

활동(Activity)명

활동 내용

관련 산출물

4.1 서버 컴포넌트 설계

아키텍처 방향, 뷰, 결정내용과 근거, 주요 기술구조 등을 포함하여 뷰를 중심으로 기술함

컴포넌트 설계모델(인터페이스와 내부설계 포함)

컴포넌트 명세서

4.1.1 인터페이스 설계

컴포넌트 명세를 기반으로 컴포넌트의 도메인 객체, DTO 객체등을 이용하여 설계

4..1.2 컴포넌트 내부 설계

컴포넌트 내부를 인터페이스, 비즈로직, 도메인 객체, 자원접근 네 가지 영역으로 분리 설계

4.1.3 컴포넌트 명세 상세화

인터페이스 설계를 기반으로 컴포넌트 명세를 상세화함

4.2 데이터 설계

컴포넌트 설계를 기반으로 데이터 설계를 수행함

코드 정의서

데이터 모델(논리모델과 물리모델 포함)

4.2.1 코드 설계

시스템 내부에서 사용할 표준 코드를 정의함

4.2.2 데이터 전환설계

이행대상 데이터가 있을 경우, AS-IS 데이터 모델을 참조하여 전환을 위한 설계를 함

4.2.3 논리 데이터 모델링

컴포넌트 내부의 도메인객체 모델을 참조하여 논리 데이터 모델을 설계함

4.2.4 물리 데이터 모델링

논리 데이터 모델을 기반으로 물리데이터 모델을 작성함

4.3 UI 설계

UI 관련 설계를 수행함

UI 설계서 (화면구조설계, 화면설계, 화면 네비게이션 포함)

4.3.1 화면구조 설계

사용자 상호작용(interaction)과 화면 레이아웃을 설계함

4.2.2 화면 설계

화면구조를 기반으로 각 화면을 상세하게 설계함

4.2.3 네비게이션 설계

상세화면을 이용하여 화면 간의 흐름을 표현하는 네비게이션을 설계함

4.4 외부 인터페이스 설계

외부 시스템과의 인터페이스를 위한 전문, 인터페이스 기능지원 코드 작상

인부 인터페이스 설계서

4.5 설계 리뷰

설계 결과를 검토하고 피드백을 반영하여 설계를 개선함

설계 검토 결과서
