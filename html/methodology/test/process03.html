테스트 구현 활동 ETVX * Option : 테스트자동화도구적용 시

ETVX(Entry, Task, Verify, eXit)로 표현되며 착수기준(Entry), 작업절차(Task), 승인(Verify)

테스트 계획 활동 개요 테스트 구현 활동은 기능 테스트 자동화 도구를 이용하여 작성된 테스트 케이스 중 전체 또는 일부를 선정하여 테스트 자동화 실행을 적용할 경우에 해당되는 활동이다.
테스트 자동화 효율성과 효과성 제고를 위하여, 테스트 자동화 스크립트 작성시 자동화 가능 여부 및 테스트 스크립트의 활용 정도를 판단해야 한다.
TAKS
  1. 자동화 대상 테스트 케이스식별
  2. 테스트 스크립트 레코딩(Recording)
  3. 테스트 스크립트 향상
  4. 테스트 데이터 준비 및 검증
  5. 테스트 스위트(Suite) 작성
  6. 테스트 자동화 스크립트 무결성 확인
  7. 테스트 스위트 검토
  8. 테스트 스위트 승인

테스트 구현 활동은 (기능) 테스트 자동화 도구를 사용하여, 자동화 테스트 수행을 적용할 경우에 수행하는 활동이다.

개발프로젝트 종료 후, 시스템이 오픈 되고 나서 시스템의 변경사항이 발생될 때 수행 되는 회귀 테스트(또는 유지보수 테스트) 수행 시, 반복적으로 사용할 테스트 케이스를 위주로 구현한다.

해당 테스트 케이스의 중요도와 실패시의 비즈니스 리스크(Risk)가 높다고 식별된 테스트 케이스를 중심으로 자동화 한다.

※ 회귀 테스트 수행을 위한 테스트 케이스 도출에 대한 상세한 방법은 별도의 "테스트 케이스 도출 가이드라인" 참고

  1. 자동화 대상 테스트 케이스 식별

    중요도가 높고 실패 시 비즈니스 리스크가 높은 테스트 케이스를 식별한다.

    반복 사용 정도가 높은 테스트 케이스를 선정한다.

    필요 시 운영 담당자와 협의하여 자동화 대상 테스트 케이스를 식별한다.

  2. 테스트 스크립트 레코딩

    전문적인 (기능) 테스트 자동화 도구를 이용하여 선정된 테스트 케이스 스크립트를 레코딩 한다.

    레코딩 된 자동화 스크립트의 이상 유무를 검증한다.

  3. 테스트 스크립트 향상

    테스트 실행 효율성을 위하여 필요시, 테스트 입력 데이터를 파라미터라이제이션 (Parameterization)한다.

    자동화 테스트 스크립트 실행 성공유무를 판단할 검증 포인트(텍스트, 비트맵, 속성, DB)를 설정한다.

    향상된 자동화 테스트스크립트 이상 유무를 검증한다.

  4. 테스트 데이터 준비 및 검증

    테스트 스크립트 실행 시 테스트 입력 값으로 사용될 테스트 데이터를 준비한다.

    준비된 데이터의 이상 유무를 확인한다.

  5. 테스트 스위트(Suite) 작성

    테스트의 효율적이고 효과적인 실행을 위하여 연관성 있는 테스트 케이스를 순서에 맞게 구성하고 테스트 스위트에 대한 속성을 정의한다.

    • 모듈 단위로 테스트 케이스를 분류한다.
    • 테스트 케이스 실행 순서를 결정한다.
    • 전문 테스터는 정의된 테스트 스위트를 확인한다.
  6. 테스트 자동화 스크립트 무결성 확인

    준비된 테스트 자동화 스크립트의 무결성을 검증한다.

  7. 테스트 스위트 검토

    테스트 설계자는 작성된 테스트 스위트를 검토((필요시 프로젝트 관리자, 운영 담당자 참여)하고 검토 의견이 있을 경우, 개선을 요청한다.

  8. 테스트 스위트 승인

    프로젝트 관리자는 작성된 테스트 스위트에 대한 승인을 프로젝트 오너로부터 획득한다.

테스트 실행 활동 ETVX

ETVX(Entry, Task, Verify, eXit)로 표현되며 착수기준(Entry), 작업절차(Task), 승인(Verify)

테스트 실행 활동 개요 테스트 실행 활동은 준비되어 있는 테스트 케이스(테스트 스위트)를 각 테스트 레벨 별 테스트 계획에 따라 실행하는 단계이다.
매뉴얼 테스트의 경우는 실행 후 테스트 케이스 별 테스트 결과 기록을 철저하게 관리하여야 하며 수동 및 자동 테스트 자산에 대한 형상 관리도 수반되어야 한다.
TAKS
  1. 테스트 환경 설정
  2. 테스트 케이스 실행
  3. 테스트 실행 결과분석
  4. 테스트 자산 개선
  5. 테스트 결과 작성
  6. 테스트 실행 결과 검토
  7. 테스트 실행 결과 승인

갱신된 테스트 결과 보고서
  1. 테스트 환경설정

    테스트 계획서에 명시된 테스트 환경을 설정한다.

    테스트 환경에는 H/W와 지원 S/W(예, 운영 시스템, 데이터베이스, 미들웨어 등)를 포함한다. 또한 다른 시스템과의 인터페이스 테스트가 필요하다면 적절한 환경이 설정되어야 한다.

    • 테스트 환경의 접근 권한을 설정한다.
    • 테스트 환경에 테스트 대상 모듈을 인스톨한다.
      필요 시, 테스트 실행을 위해 추가적으로 요구되는 지원 프로그램(테스터 스텁(Test Stub), 테스트 드라이버(Test Driver), 시뮬레이터, DB 확인 SW 등) 구축
    • 테스트를 위해 작성해 놓은 최소 테스트 데이터를 준비한다. 실제 데이터를 이용해 테스트 할 경우, 운영 환경에서 데이터를 다운받아 테스트 환경에 배치한다.
      필요시, 테스트 자동화 도구 준비

    ※ 단위 테스트는 일반적으로 별도의 테스트 환경을 준비하지 않고 개발 환경에서 테스트를 실행한다.

  2. 테스트 케이스 실행

    해당 테스트 케이스를 실행한다. 자동화 도구를 사용할 경우, 테스트 자동화 스크립트를 실행한다.

    테스트를 수행하면서 발생하는 사항들을 (테스트 케이스 명세서 상에) 로그(Log)로 기록

    테스트 케이스 명세서에 기술된 테스트 기준 및 예상결과를 실제 테스트 결과와 비교, 일치하지 않거나 문제가 발생한 경우 해당 사항은 결함관리대장 또는 SR시스템에 기록한다.

    실제 결과가 예상 결과와 일치할 경우 결과를 ‘Pass’로, 일치하지 않을 경우 ‘Fail’로 테스트 케이스 명세서에 기록한다. 작성된 테스트 케이스를 실행하지 못했을 경우 ‘No Run’으로 기록한다.

    ※ 테스트 결과 기록은, 결함관리 활동 및 해당 테스트 가이드라인을 참고하여 수행한다.

  3. 테스트 실행결과 분석

    Fail로 기록된 테스트 실행 결과를 분석하여 사용자의 단순한 에러인지 결함인지 판단하여 결함 관리대장에 기록한다.

    발견된 결함을 결함관리대장 양식을 이용하여 기록한 후 결함 내역, 결함 분류, 결함 발생 단계, 결함 심각도 등의 속성을 입력한다.

    테스트 케이스 명세서에 기록된 예상 결과를 기반으로 테스트가 통과되었는지, 실패되었는지 결정.

    테스트 실행 및 결함 조치에 소요된 자원 내역을 기록

    실패한 테스트 각각에 대해 결함을 분석하고 기록

    ※ 결함 기록 및 관리는 결함기록관리 활동 상세 Task를 참조하여 수행한다.

  4. 테스트 자산 개선

    테스트를 수행하면서 변경이 발생된 테스트 케이스, 테스트 요구사항, 테스트 데이터 등의 테스트 자산을 현행화한다.

  5. 테스트 결과 작성

    테스트 케이스 명세서에 해당 테스트 케이스 실행 결과를 작성한다.

    이때의 테스트 결과 보고는 최종 보고서가 아닌 단순한 테스트 케이스 실행에 대한 결과 보고이다.

    이때 작성된 결과는 향후 최종 결과 보고서 성에 활용된다.

    단위 테스트의 경우, 일반적으로 개발자가 단위 테스트 결과 보고를 작성한다.

  6. 테스트 실행 결과 검토

    테스트 설계자는 작성된 테스트 결과 및 결함조치 내역을 취합하여 프로젝트 관리자와 검토 및 협의를 수행한다.

    검토 의견이 있을 경우, 테스트 설계자는 합의하여 검토 의견을 반영한다.

  7. 테스트 실행 결과 승인

    프로젝트 관리자는 작성된 테스트 케이스에 대한 승인을 프로젝트 오너로부터 획득한다
