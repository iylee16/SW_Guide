도구 소개

도구명 OSRMT
(https://sourceforge.net/projects/osrmt/) 		라이센스 GPL2
소개
  • 요구사항 관리 도구이다.
  • 요구사항을 정의하고 전체 개발 과정에서 요구사항을 추적할 수 있도록 지원한다.
특징
  • 요구사항을 SDLC 별로 구분하여 정리 가능
  • 요구사항 간의 의존성을 관리 가능
  • 요구사항 내용을 문서화 가능
주요 기능
  • 요구사항 등록
  • 요구사항 추적
  • 요구사항 문서 생성
실행 환경 Windows, Linux, MacOS 최신 버전 DBMS, Java 8.0이상
카테 고리 요구사항 관리 최신 버전 v1.5 (2013.05)
관련 도구
  • JFeature, JRequisite, RequirementHeap

설치 환경

  • Windows의 경우 아래와 같이 Command창에서 ‘java –version’을 실행하여 JDK 설치 여부를 확인한다.
  • Java 8.0 이상을 권장한다.
  • MySQL, Oracle, MS Access, SQLServer 와 같은 DBMS가 설치되어 있어야 하며 Running 상태여야 한다.
  • MySQL의 경우 아래와 같이 MySQL Workbench에서 Server Status를 보고 실행 상태가 Running임을 확인한다.

1. 다운로드

  • http://sourceforge.net/projects/osrmt/files/?source=navbar 에서 설치 파일을 다운 받을 수 있다.
  • 최신 버전이 표시된 곳을 선택하여 설치 파일을 저장한다.

2. 설치

  • 다운받은 압축파일(zip)을 풀면 osrmt15.jar 파일이 생성된다.
  • Command창에서 ‘java -jar osrmt15.jar’을 실행한다.
  • 언어 선택 창이 열리면 OK를 선택한다.
  • 라이센스 동의 및 설치 경로 결정 후 Next 선택한다.
  • 필요한 설치 패키지를 선택한다.
  • 예시에서는 2tier 프로그램을 설치할 것이며, Base, Client, Docs를 선택하도록 한다.
  • 3tier 프로그램 설치를 원하는 경우에는 웹서버를 이용하는 것으로 Server 패키지를 같이 설치해야 한다.
  • 아이콘 생성 설정을 확인하고 Next를 선택한다.
  • 파일 설치가 진행되는 것을 확인할 수 있다.
  • 파일 설치가 완료되고 Next 버튼이 활성화되면 선택한다.
  • 설치가 성공적으로 완료되었다는 메시지를 확인하고 Done 을 선택하면 Installer 창이 닫힌다.
  • 실행 아이콘을 생성된 것을 확인할 수 있다.
  • Program Files – osrmt 폴더에 설치된 것을 확인할 수 있다.

3. 실행

  • SRMT Client 아이콘을 더블 클릭하여 실행한다.
  • Log On 창에 Username과 Password를 입력하고 OK를 선택한다.
  • 예제로 사용된 사용자는 DEMO/demo이다.
  • Log On이 성공하여 실행된 초기 화면은 다음과 같다.

프로젝트 관리 – 프로젝트 등록

메뉴 File – New Product를 선택한다.

New Product 창에서 프로젝트명을 입력하고 OK를 선택한다.

메뉴 File – Open을 선택한다.

Open Product 창에서 Product를 선택하고 OK를 선택한다.

신규 등록 항목 선택

탐색창에서 등록할 항목을 선택하고 New 버튼을 선택하면 해당 항목을 입력할 수 있는 창이 열린다.

Feature, Requirement, Design, Implementation, TestCase를 선택하여 입력할 수 있다.

특성(Feature) 등록

Feature, Version, Description을 입력하고 Apply를 선택하면 등록된다.

최초 등록의 경우 Priority, Status 는 기본값을 그대로 둔다.

요구사항(Requirement) 등록

Requirement, Version, Priority, Status, Description을 입력한다.

추가적으로 Complexity, Effort, Category정보를 더 입력할 수 있다.

Apply를 선택하면 등록된다.

설계(Design) 등록

Design, Priority, Status, Version, Description을 입력한다.

추가적으로 Category 정보를 입력할 수 있다.

Apply를 선택하면 등록된다.

구현(Implementation) 등록

Source Code, Status를 입력한다.

추가적으로 Category 정보를 입력할 수 있다.

Apply를 선택하면 등록된다.

테스트케이스(TestCase) 등록

Test Case를 입력하고 Purpose, Prerequisite, Test Data, Steps, Expected, Notes 등의 추가 정보를 입력 후 Apply를 선택하면 등록된다.

Steps에 대한 정보 입력은 다음 페이지에서 설명한다.

Add Row를 선택하여 Step을 추가한다.

Step 내용을 입력한 후 Apply를 선택하면 등록된다.

Step 항목의 순서는 Move Up, Move Down 버튼으로 이동할 수 있다.

Dependency 등록

모든 항목(Feature, Requirement, Design, Implementation, TestCase)에서 Dependencies 탭을 선택하면 dependency를 등록할 수 있다.

Dependescies 탭 선택 후 마우스 오른쪽 선택을 하여 New를 선택한다.

Dependency를 설정할 대상 항목을 선택한 후 OK를 선택한다.

Dependent on에 선택된 항목이 추가된 것을 확인할 수 있다.

예제 소개

일정정보에 대해 Excel과 연동이 필요한 소프트웨어의 요구사항을 관리하는 간단한 예제이며 다음과 같은 시나리오로 진행된다.

  1. 해당 소프트웨어의 필요 특성에 대해 Feature를 등록한다.
  2. Excel과 연동하는 기능적 요구사항에 대해 Requirement를 등록한다.
  3. 구현해야 할 모듈을 Implementation으로 등록하고 TestCase를 생성한다.
  4. Traceability Matrix를 활용하여 등록된 Feature, Requirement, Implementation, TestCase 항목들 간의 dependency를 설정한다.
  5. Feature가 변경되는 경우를 가정하여 항목 간의 Change Impact를 Trace하는 기능을 확인한다.
