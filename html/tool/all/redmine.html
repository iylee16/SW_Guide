도구 소개

도구명 Redmine
(http://www.redmine.org/) 		라이센스 GNU GPL v2.0
소개 Ruby on Rails 기반의 프로젝트 관리와 버그 추적 기능을 제공하는 도구이다.
특징
  • 유연한 이슈 및 버그 추적
  • 간트 차트와 달력기능 제공
  • 다중 프로젝트 지원
  • 형상관리 소프트웨어와의 통합
주요 기능
  • 프로젝트 관리, 이슈관리, 간트 차트
  • 이메일 알림
  • 다중 프로젝트 관리, 프로젝트 별 위키 페이지
  • 형상관리 소프트웨어 연동
실행 환경 Windows, Linux, MacOS 사전 설치 도구 DBMS, WebServer
카테 고리 프로젝트 관리 최신 버전 v3.4.6 (2018.06)
관련 도구
  • JFeature, JRequisite, RequirementHeap

주요 기능

  • 자원관리, 이슈관리
  • 프로젝트 위키 페이지
  • 자체 문서 관리 기능, 형상관리 연동 기능

설치 환경

Redmine은 웹 기반의 프로젝트 관리 기능을 제공하는 웹 애플리케이션이다.

  • 웹서버와 DBMS가 필요하고 Redmine은 Ruby로 제작되어 Ruby도 설치해야하는 등 구성요소가 많고 설치가 매우 복잡하다.
  • Redmine 홈페이지에서는 어려운 개별 설치의 대안으로 아래의 두가지 Third-party Bundle을 안내한다.
    ( http://www.redmine.org/projects/redmine/wiki/Download#Third-party-Redmine-bundles)
    • TurnKey Redmine Appliance
    • BitNami Redmine Stack

본 지침에서는 각 구성 요소를 개별적으로 설치하지 않고, Windows 환경에서도 한번에 쉽게 설치할 수 있는 Bitnami Redmine Stack을 기준으로 설치 방법 설명한다.

설치 단계

1. BitNamie Redmine Stack 다운로드

https://bitnami.com/stack/redmine/installer에 접속하여 Download for windows를 선택한다.

  • OS에 맞는 Installer가 자동으로 표시되며, 본 지침에서는 윈도우 Installer 선택한다.

    ※ 회원가입 없이 No. Thanks 선택을 통해 다운이 가능하다.

2. BitNami Redmine Stack 설치

다운로드 받은 파일을 실행한다.

  • 언어 선택 : 한국어
  • Anti-Virus 관련 질문에 ‘예’를 선택한다.

다운로드 설치 마법사

  • ‘다음’ 선택한다.
  • 구성요소 선택 : ‘다음’을 선택한다.

설치 경로

  • ‘다음’을 선택하여 기본값으로 지정된 위치에 설치한다.

Admin 계정 생성

  • Admin 계정에 대한 정보를 입력한다.
  • 여기서는 로그인 계정명은 admin, 패스워드는 redmine으로 설정한다.

데이터 설정 언어

  • 기본값으로 ‘한국어’가 선택되어 있음을 확인하고 ‘다음’을 선택한다.

SMTP 설정

  • 사용하시겠습니까? 항목을 체크한다.
  • 기본 이메일 서비스 제공자 Gmail 선택 후 ‘다음’을 선택한다.
  • Gmail 계정 정보를 입력한다.
  • 계정 정보가 설정 파일에 그대로 저장되므로 개인적인 메일 계정 정보를 사용하지 않을것을 권장한다.

BitNami 클라우드 호스팅

  • 본 가이드에서는 클라우드 호스팅은 사용하지 않으므로 선택을 해제하고 ‘다음’ 선택한다.

설치 준비 완료

  • ‘다음’을 선택한다.

설치 완료

  • Redmine 어플리케이션 구동을 선택하고 ‘마침’을 선택한다.

3. BitNami Redmine Stack 설치 확인

실행 화면

  • 아래와 같이 localhost에서 Bitnami 화면이 표시되고 BitNami Redmine Stack이 로딩된다.
  • Access BitNami Redmine Stack을 선택한다.

실행 화면

  • 아래와 같이 http://localhost/redmine 에서 Redmine이 정상적으로 구동된다.

사용자 관리

로그인 및 사용자 관리

  • 설치 시 설정했던 관리자 계정으로 로그인한다.
  • 상단메뉴 → ‘관리’ → ‘사용자’를 선택한다.

사용자 추가

  • ‘새 사용자’를 선택한다.

항목을 작성 후, ‘만들기’를 선택한다.

  • 계속해서 추가하려면 ‘만들고 계속하기’를 선택한다.

오른쪽 메뉴의 ‘사용자’를 선택한다.

  • 사용자가 만들어 졌음을 알 수 있다.

등록한 메일로 가입 확인을 할 수 있다.

사용자 삭제

  • 사용자 메뉴에서 삭제할 사용자의 ‘삭제’ 버튼을 누른다.

프로젝트 관리

프로젝트 관리

  • ‘관리’ → ‘프로젝트’를 선택한다.
    • 현재 프로젝트 목록 확인 및 관리가 가능하다.
  • 퇴근 전에 현장 Code를 형상관리 도구에 저장(commit)함으로써 source Code에 대한 자동 Build를 수행할 수 있다.

프로젝트 생성

  • ‘새 프로젝트’를 선택한다.
  • 항목을 채워넣은 후, ‘만들기’를 선택한다.
    • 모듈 선택에서 사용하고자 하는 기능들을 선택 가능하다. (생성 후, 설정에서도 관리 가능)
  • ‘개요’를 선택한다.
    • 생성된 프로젝트의 개요를 확인 할 수 있다.

프로젝트 삭제

  • ‘관리’ → ‘프로젝트’를 선택한다.
  • 삭제할 프로젝트의 ‘삭제’를 선택한다.
  • ‘예’ 체크박스 선택 후, 삭제 버튼을 선택한다.
    • 삭제 후, 프로젝트가 삭제되었는지 확인한다.

프로젝트 담당자 설정

  • 초기 화면에서 ‘일정 관리 프로그램’ 프로젝트를 선택한다.
  • ‘설정’ → ‘담당자’를 선택한다.
  • 오른쪽의 ‘새 담당자’에서 담당할 사람과 역할을 선택 후 ‘추가’를 선택한다.

담당자가 추가 되었음을 확인할 수 있다.

이슈 관리

이슈 확인

  • 본 예제에서는 앞에서 만들었던 ‘일정 관리 프로그램’ 프로젝트를 바탕으로 설명한다.
  • ‘일감’을 선택한다.
    • 현재 진행중인 이슈를 확인 할 수 있다.

이슈 등록

  • ‘새 일감 만들기’를 선택한다.
  • 해당 항목들을 채운 다음 ‘만들기’를 선택 한다.
  • 등록 후, ‘일감’ 메뉴를 선택한다.
    • 이슈가 추가됨을 확인 할 수 있다.

작업 기록

  • 담당자의 아이디(홍길동, test_1)로 로그인 후, ‘일감’ 메뉴에서 추가된 이슈를 선택한다.
  • ‘작업시간 기록’을 선택한다.
  • 항목을 작성한 후, ‘만들기’를 선택한다.
    • 시간은 ‘시’ 단위로 나타낸다.

이슈 해결

  • 해당 이슈에서 ‘수정’을 선택한다.
  • 진척도를 100%로 설정, 상태를 ‘완료’로 선택. 이후 ‘확인’을 선택한다.
    • 다른 상태(신규, 해결, 진행, 의견, 완료)의 선택이 가능하다.

‘일감’ 메뉴에서 검색 조건을 모두로 선택 후, 적용한다.

  • 완료된 이슈까지 표시된다.

이슈 요약

이슈 요약

  • ‘일감’ 메뉴에서 오른편의 ‘요약’을 선택한다.
  • 요약된 이슈 정보를 확인 할 수 있다.

프로젝트 개요

프로젝트 개요를 볼 수 있다.

  • 메인 메뉴에서 ‘개요’를 선택한다.

작업 내역

작업 내역을 한번에 볼 수 있다.

  • 메인 메뉴에서 ‘작업내역’을 선택한다.

간트 차트

프로젝트 진행 상황을 간트 차트를 통해 확인 할 수 있다.

  • 메인 메뉴에서 ‘Gantt 차트’ 선택한다.

달력

프로젝트 진행 상황을 달력을 통해 확인 할 수 있다.

  • 메인 메뉴에서 ‘달력’을 선택한다.

뉴스

뉴스 등록 및 열람 가능

  • 메인 메뉴에서 ‘뉴스’를 선택한다.
    • 권한을 가지고 있는 사용자만 뉴스 등록 가능하다.

문서

문서 등록 및 열람 가능

  • 메인 메뉴에서 ‘문서’를 선택한다.
    • 권한을 가지고 있는 사용자만 문서 등록이 가능하다.

위키 기능

프로젝트 내에서 위키 기능을 제공한다.

  • 메인 메뉴에서 ‘위키’를 선택한다.

파일

프로젝트에서 사용되는 파일을 관리 할 수 있다.

  • 메인 메뉴에서 ‘파일’을 선택한다.

환경 설정

Redmine의 전체적인 환경 설정이 가능하다.

  • 최상단 메뉴에서 ‘관리’를 선택한다.
    • 하위 메뉴에서 상세한 설정이 가능하다.

프로젝트 환경 설정

프로젝트의 환경 설정이 가능하다.

  • 메인 메뉴에서 ‘설정’을 선택한다.
    • 만약 저장소와의 연동이 필요하면, 이 메뉴에서 설정한다.

이메일 알림 기능

프로젝트 변경(이슈, 뉴스 및 변경내역)시 이메일 알림 기능을 제공한다.

  • 프로젝트 생성, 사용자 생성 시에 이메일 알람 설정을 하면 다음과 같이 이메일을 받을 수 있다.

주요 추가 기능

role-based issue custom field visibility

  • 자주 사용하는 query(한국어 버전에서는 ‘검색 양식’) 생성 시 해당 query의 visibility를 role별로 지정이 가능하다.

Gantt chart : 이슈 관계 및 진척 그래프가 표시된다.

파일 첨부를 Drag&Drop을 통해 AJAX 방식 수행 및 업로드 진척도가 표시된다.

새 사용자 정의 항목 입력 항목을 추가한다.

  • 기존 버전에서 형식, 이름, 길이, 정규식, 기본값만 설정 가능하였으나 2.5.0 이후 설명과 URL링크 값이 추가되었다.

구성원으로 외부인원 추가 기능

  • 프로젝트의 구성원으로 사용자와 그룹 이외에 불특정 인원도 참여할 수 있도록 설정하는 기능을 추가되었다.

메일 알림 조건 추가

  • 일감 수정 시 Assignee update를 추가하여 설정 기능이 추가되었다.

예제 소개

프로젝트 시나리오

  • A회사는 일정관리 어플리케이션 개발을 의뢰 받았다.
  • 프로젝트 참여 인원은 3명이다.
  • 현재 단계는 개발단계. 그러나 갑작스런 고객의 요구사항 추가 요청이 발생한다.
  • 관리자는 발생한 이슈의 해결 과정을 팀원에게 메일로 알려주기로 한다.
  • 이 이슈의 담당자는 ‘홍길동’ 개발자이다.
  • 개발자는 추가 요청을 해결하고 이를 관리자에게 알렸다.

추가 요청 사항

일정 내용을 E-mail로 보내는 기능을 요구했다.

팀원 계정 생성

관리자 계정으로 시작한다.

‘관리’ → ‘사용자’에서 관리자를 포함한 3명의 사용자를 추가했다.

프로젝트 생성

‘관리’ → ‘프로젝트’에서 새 프로젝트를 추가한다.

프로젝트 내 ‘설정’ → ‘담당자’에서 새 담당자를 추가한다.

이슈 등록

메뉴의 ‘새 일감 만들기’에서 이슈를 추가했다.

  • 담당자는 ‘홍길동’으로 지정한다.

이슈 등록 이메일 확인

개발자 ‘홍길동’은 자신의 메일을 확인한다.

이슈 해결

개발자 ‘홍길동’은 등록된 이슈를 해결하고 이를 Redmine에 등록한다.

  • ‘홍길동’ 계정으로 접속 후, 이슈 상태를 확인한다.

이슈 상태를 ‘해결’로 변경, 진척도를 변경 후 수정한다.

이슈 해결 이메일 확인

관리자는 이슈가 해결되었다는 메일을 확인 할 수 있다.

  • 관리자 이메일 계정으로 접속하여 확인한다.

이슈 종결

관리자는 해결된 이슈를 확인하고, 이슈의 상태를 ‘완료’로 설정한다.

  • 관리자의 아이디로 로그인 하여 해당 이슈를 선택한다.
  • ‘수정’을 선택하여 상태를 ‘완료’로 설정, 진척도를 100% 설정 후 ‘확인’을 선택한다.

이슈 종결 이메일 확인

개발자는 이슈가 종결되었음 알리는 메일을 확인한다.

  • 개발자의 이메일 계정으로 접속하여 메일을 확인한다.
