도구 소개

도구명 SonarQube
(http://www.sonarqube.org/) 		라이센스 GNU Lesser GPL v3
소개 SonarQube는 Java 기반의 정적분석 도구로써, 다양한 품질 항목들에 대해 분석하여 사용자에게 결과를 제공한다.
특징
  • 여러 분석 툴을 통합하여 여러 항목들을 한번에 검사 가능
  • 웹 페이지에서 분석 결과들을 손쉽게 확인 가능
  • 플러그인을 이용하여 다양한 기능 제공
주요 기능
  • 코드 품질 대쉬보드
  • 코드 중복, 코딩 표준, 커버리지 부족, 잠재 버그, 복잡도, 문서화, 설계에 대한 품질 척도 표시
  • Java, C#, C/C++, JavaScript, PHP 등 20여가지 언어 지원
  • CI(Continuous Integration) 도구와 통합
  • 60 여가지의 플러그인 제공
실행 환경 Windows, Linux, MacOS 사전 설치 도구 JDK 1.6 이상
카테 고리 코드 품질 관리 최신 버전 v7.3 (2018.08)
관련 도구
  • CheckStyle (http://checkstyle.sourceforge.net/)
  • PMD (https://pmd.github.io/)
  • Splint (http://www.splint.org/)

주요 기능

SQALE(Software Quality Assessment based on Lifecycle Expectations) 등급 및 기술적 부채 비율 표시

품질 지표를 한 눈에 볼 수 있는 SW Visualization 대쉬보드 제공

품질 게이트 지정 및 점검 수준 정의

설치 환경

windows, Linux, MacOS 별로 별도의 설치 파일 다운 및 설치

사전 JDK 1.7이상의 도구 설치 필요(https://www.oracle.com/technetwork/java/javase/downloads/index.html)

※ JDK 설치 후 시스템 환경 변수 등록 필요

시스템 변수 명 : Path, 변수값 : JDK 설치 폴더\bin

설치 단계

1. SonarQube 설치

SonarQube를 다운받는다.

  • 다운로드 사이드 : https://www.sonarqube.org/downloads/

다운받은 파일을 압축 해제한다.

  • 본 예제에서는 D:\SonarQube에 압축 해제한다.

※ D:SonarQube를 <sonarqube-home>으로 지정한다.

DB 접속 설정

  • DB 접속 정보는 <sonarqube_home>/conf/sonar.properties 파일에서 관리한다.
    • 내장 DB를 사용하는 경우 수정할 필요가 없다.
    • 여기서는 내장 DB를 사용한다.

Web Server 설정

  • sonarqube_home>/conf/sonar.properties 파일을 수정한다.
    • sonar.web.host, sonar.web.context, sonar.web.port를 주석 해제하고 편집한다.

SonarQube 서비스 시작

  •  <sonarqube_home>/bin/<사용자OS>/ 폴더에서 StartSonar.bat 파일을 실행한다.
  • VM 관련 에러가 날 경우 <sonarqube_home>/conf/wrapper.conf 파일 내의 wrapper.java.initmemory와 wrapper.java.maxmemory의 값을 128, 256으로 지정한다.

SonarQube 웹 브라우저 접속 확인

  • 주소 : http://localhost:9000/sonarqube
    • 접속이 되면 Sonar 설치 완료 (최초 로그인 ID/PW : admin / admin)

2. SonarQube Runner 설치

Sonar에서는 분석을 위한 여러 클라이언트를 제공한다.

  • Sonar runner : 권장되는 기본 클라이언트이다.
  • Maven : Maven 기반으로 작성된 프로젝트에 적합하다.
  • Ant Task : Ant 기반으로 작성된 프로젝트에 적합하다.

본 예제에서는 제작사에서 권장하는 Sonar runner를 설치한다.

  • SonarQube의 다운로드 사이트에서 Sonar Runner(Scanner) 다운로드 받는다.
    • http://www.sonarqube.org/downloads/

다운로드 및 압축 해제

  • SonarQube Runner는 별도의 설치 과정 없이 압축 해제만으로 설치가 완료한다.

환경변수 등록

  • 설치한 폴더의 bin 폴더를 path에 환경변수를 등록한다.

Sonar runner 환경 설정

  • “설치폴더\conf\sonar-runner.properties” 파일을 Sonar에서 세팅한 값과 맞춘다.
    • 본 예제에서 사용할 내장 server 사용을 위해 두 번째 줄을 수정 “#sonar.host.url=http://localhost:9000” -> “sonar.host.url=http://localhost:9000/sonarqube”

Sonar 소개

Sonar는 Checkstyle, PMD, FindBugs, Cobertura등의 분석 툴들을 이용하여 소스의 정적 분석을 실시하고, 이를 웹 브라우저를 통하여 전체적인 분석 결과를 보여주는 도구이다.

Sonar가 제공하는 분석 결과는 다음과 같다.

  • Duplicate code
  • Coding standards
  • Code coverage
  • Complex code
  • Potential bugs
  • Comment
  • Design and Architecture

기본적으로는 Java 언어를 분석할 수 있지만, 추가 플러그인을 통하여 다른 대부분의 언어에 대해서도 검사가 가능하다.

  • C, C#, PHP, Flex, Groovy, JavaScript, Python, PL/SQL, Cobol, Visual Basic 6

Sonar 자체는 분석 결과를 보여주는 툴. 실제 분석은 Sonar 내의 Plug-in을 통하여 분석한다.

  • 기본적으로 제공되는 분석 툴
    • PMD : 소스코드의 표준 준수 분석
    • FindBugs : 자바 바이트 코드를 분석하여 정적 분석
    • CheckStyle : 코딩 스타일 체크
    • Cobertura : 테스트 커버리지 체크
    • Surefire : 유닛 테스트

Sonar에서 분석 결과를 보기위해 분석을 위한 Client의 종류는 다음과 같다.

  • Sonar runner : 권장되는 기본 클라이언트
  • Maven : Maven 기반으로 작성된 프로젝트에 적합
  • Ant Task : Ant 기반으로 작성된 프로젝트에 적합

Sonar 메뉴

Sonar의 메인 화면의 Dashboard

  • Project 에서 분석된 프로젝트들의 목록을 볼 수 있다.

Overview : 프로젝트 내의 Dashboard

  • 분석 결과의 전체적인 내용을 요약하여 보여준다.

Components 메뉴

  • 프로젝트를 컴포넌트 별로 시각화하여 보여준다.

Components Drill-Down

  • 폴더 – 소스파일 로 Drill-Down하여 소스코드까지 확인 가능하다.

Issues 메뉴

  • 현재 프로젝트에서의 활동 사항을 보여준다.

Issues 하위 메뉴

  • 분석 결과 중, 수정 사항이 필요한 부분들을 계층별로 탐색하여 보여준다.

Issue Report 메뉴

  • 이슈에 대한 처리 상태, 중요도 등에 따라 이슈에 대한 통계를 확인할 수 있다.

Time Machine 메뉴

  • 프로젝트 현황을 버전 및 기간별로 보여준다.

Sonar 환경 설정

Sonar 전체 환경설정

  • 관리자 계정으로 접속하여 상단의 ‘Settings’ 메뉴를 선택한다.

사용자 관리

  • 사용자 등록 및 삭제는 ‘Security’ → ‘Users’ 메뉴 선택한다.
    • 오른쪽의 ‘Add new user’를 이용하여 사용자 추가가 가능하다.

Email 서버 설정

  • Sonar는 smtp를 이용하여 사용자에게 이메일 수신 기능을 제공한다.
    • ‘Configuration’ → ‘General’ → ‘Email’ 메뉴에서 설정

Plug-in 관리

  • Sonar는 자체에서 Plug-in을 손쉽게 설치 및 관리하는 기능을 제공한다.
    • ‘System’ → ‘Update Center’ 메뉴를 선택한다.

프로젝트 내 환경설정

  • 해당 프로젝트에서 상단의 ‘Settings’ 메뉴에서 제공한다.
    • 프로젝트 내에서 분석 정도 및 다양한 항목에 대해 설정이 가능하다.

계획 수립

Sonar는 이슈들을 계획으로 그룹화하여 관리가 가능하다.

  • 프로젝트 내의 ‘Settings’ → ‘Action Plans’을 선택한다.
    • 현재 계획들의 목록 및 진행 사항을 볼 수 있다.

계획 추가

  • 오른편의 ‘Add New Action Plan’을 선택하여 항목을 채운 뒤 ‘Create’를 선택한다.
    • 새 계획이 추가된다.

이슈 관리

이슈 확인

  • Issues 페이지에서 분석된 소스 파일을 선택한다.
    • 해당 코드와 함께 문제점에 관한 내용을 보여준다.

이슈 할당

  • 할당할 이슈에 중요도, 상태, 담당자, Action plan, Comment 등을 입력한다.

할당 받은 이슈 확인

  • Issue 메뉴의 Assignee – Me 에서 자신에게 할당된 이슈를 확인 할 수 있다.
    • 해당 제목을 선택하면 상세 정보 확인이 가능하다.

중요도 변경

  • Blocker, Critical, Major, Minor, Info로 처리 중요도 수준을 변경 할 수 있다.

이슈 상태 변경

  • Confirm, Resolved as Fixed, Resolved as false positive, Resolve as won’t fix로 처리 결과에 따라 상태를 변경 할 수 있다.

Sonar Runner(Scanner) 사용

Sonar runner 설정 파일을 생성한다.

  • 분석하고자 하는 프로젝트 폴더에 “sonar-project.properties” 파일을 생성한다.
    • 본 예제에서는 Adroid-BitmapCache\library 폴더에 생성한다.

sonar-project.properties 파일 내용

  • Project에 관한 Key, name, Version 내용
  • 실제 Source code가 포함되어 있는 곳
  • 테스트 코드의 위치
  • 바이너리 파일의 위치
  • 라이브러리 파일의 위치
  • 언어 종류
  • 추가 파라미터

설정 파일을 작성 한 후, 커맨드 창에서 다음을 입력한다.

  • sonar-runner

분석이 완료되면 Sonar 웹 페이지에서 확인 가능하다.

예제 소개

Open Source인 Android-BitmapCache 프로젝트를 정적 분석한다.

Sonar 클라이언트는 Sonar-runner를 사용한다.

문제점을 개발자에게 할당 및 해결한다.

인원은 관리자 1명, 개발자 1명이다.

Sonar-runner(Scanner) 사용

sonar-project.properties 파일을 작성한다.

  • ProjectKey : my:adnroid_bitmapcache
  • ProjectName : Android-bitmapCache
  • ProjectVersion : 1.0
  • 소스파일 및 라이브러리 파일 위치를 지정한다.
  • 사용 언어는 Java이다.

Sonar-runner를 이용하여 분석 실행

  • 해당 폴더에서 커맨드창을 열어 다음 명령어 실행한다.
    • sonar-runner

분석 결과 확인

http://localhost:9000/sonarqube 으로 접속하여 프로젝트 생성을 확인한다.

  • Android-BitmapCache 프로젝트가 생성됨을 확인이 가능하다.

프로젝트 내 Dashboard 확인

  • 프로젝트의 검사 결과를 확인 할 수 있다.
    • 수정 사항이 56가지임을 확인 가능하다.

사용자 등록

사용자(개발자) 계정을 등록한다.

  • Settings → Security → Users에서 등록한다.

이슈 할당

Issues 메뉴에서 예시로 수정 사항 하나를 선택. 여기선 ‘Remove those useless parentheses’ 를 선택한다.

오른쪽의 화살표를 눌러 세부 사항을 확인한다.

문제점 확인

  • 쓸모 없는 괄호가 사용되었음을 알 수 있다.
