도구 소개

도구명 Jenkins (https://jenkins.io/) 라이센스 MIT
소개
  • CI (지속적 통합) 도구
  • 빌드, 배포 등 반복되는 작업을 모니터링하는 도구
특징
  • 쉬운 설치 및 웹 기반으로 된 쉬운 UI 제공
  • 여러 project 동시 Build 제공
  • 다양한 언어 및 OS 지원
주요 기능
  • 지속적인 자동화 빌드
  • 지속적인 자동화 테스트
  • 잡 스케줄링
  • 자동화된 배포 관리
실행 환경 Windows, Linux, MacOS 사전 설치 도구 Java 8 이상
카테 고리 빌드/배포 최신 버전 v2.138.2 (2018.10)
관련 도구
  • Ant, maven, Git, jUnit, Clover

주요 기능

  • 형상 관리도구와 연동하여 소스 가져오기
  • 빌드 및 배포 자동화
  • 자동화를 위한 다양한 플러그인 제공

설치 환경

Windows의 경우 아래와 같이 Command창에서 ‘java –version’을 실행하여 설치된 JDK 설치 여부를 확인한다.

Java 8.0 이상을 권장한다.

Jenkins는 홈페이지에서 war 형태로 배포되어 별도의 웹 애플리케이션 서버(WAS)에 배포하여 운영하지만 본 자료에서는 쉬운 환경 구성을 위해, Jenkins 홈페이지에서 윈도우 운영체제 사용자를 위해 제공하는 WAS포함 Jenkins 패키지로 설치해서 진행한다.

설치 단계

1. Jenkins 설치 - Package

Jenkins 홈페이지에서 설치파일을 다운로드한다.

  • https://jenkins.io/
  • 화면의 download를 선택하여 설치 파일을 다운로드한다.

다운로드 한 zip 파일의 압축을 푼다.

Setup 파일을 선택하여 설치를 시작한다.

2. 서비스 기동

설치가 완료되면 자동으로 기본 브라우저가 실행되고 Jenkins가 구동된다.

Windows의 ‘ 컴퓨터관리>서비스 및 응용프로그램>서비스 ‘ 에서 Jenkins를 확인할 수 있으면 서비스로 정상적으로 등록된 것이다.

상태가 ‘시작됨’이고 시작 유형이 ‘자동’이면 부팅 시에 자동으로 실행된다.

Jenkins 설치 시 8080을 기본 포트로 사용한다.

시스템에서 8080 포트를 이미 사용하고 있는 경우는 Jenkins 설치 디렉토리 > Jenkins > jenkins.xml

파일에서 포트 설정을 변경한다.

jenkins.xml을 편집기나 노트패드로 열어 -httpPort=8080 포트를 다른 포트로 변경한다.

Continuous Integration이란?

Jenkins는 Continuous Integration(지속적 통합)을 지원하는 도구로서 여러 개의 project 또는 File들의 Build를 편리하고 쉽게 할 수 있는 환경을 제공한다.

Commit에 따른 자동 Build 제공 : source Code가 형상관리 도구에 commit이 되었을때마다 CI 도구가 이를 감지하고 자동으로 Build를 수행하도록 설정한다.

일일 check out과 Build : 개발자가 출근 후 형상관리 도구에서 최신 Code를 내려 받고, 출근 전에 현재 Code를 형상관리 도구에 저장함으로써 Source Code에 자동 Build를 수행한다.

Jenkins 의 주요 기능

Source Code의 일관성을 유지한다.

  • CI도구를 설정하기 위해서는 기본적으로 형상관리 도구가 필요하다.
  • 대표적인 형상관리 도구는 Subversion, CVS, Git 등이 존재한다.
  • CI도구는 이 형상관리 도구로부터 Project source의 Code를 Check out 받아서 Build를 수행한다.

자동 Build

  • Build가 이루어지는 시점을 정하는 두가지 방법.
  • Commit에 따른 자동으로 Build.
  • Source Code가 형상관리도구에 Commit이 되었을 때마다 CI 도구가 이를 감지하고 자동으로 Build를 수행하도록 설정한다.
  • Build 시간이 길 경우 Build가 적체되는 현상이 발생할 수 있다.
  • 정기적인 Build
  • 일정 시각을 정해서 Build를 하는 방법이다.
  • Build schedule이 정해져 있기 때문에 개발자들이 Commit에 대한 Schedule을 관리할 수 있고 Build 시간이 오래 걸리는 대규모 Build에도 적정하다.

자동 테스팅 코드 기동

  • Build 과정에 테스트를 포함한다.
  • 주기적인 Build 과정에 테스트를 포함시켜서 Build와 더불어 테스팅을 자동으로 수행한다.
  • Software의 품질 향상 기여한다.

일일 Checkout과 Build

  • 개발자가 출근 후 형상관리도구에서 최신 Code를 내려 받고(Check out),
  • 퇴근 전에 현장 Code를 형상관리 도구에 저장(commit)함으로써 source Code에 대한 자동 Build를 수행한다.

Web 개발을 위한 개발환경 하에서 Git에 저장되어 있는 소스를 Jenkins 에서 읽어와서 Maven으로 빌드를 수행하고 빌드된 파일을 WAS에 배포한다.

Maven과 Git이 설치되어 있다고 가정하고, GitHub의 저장소를 복제해서 로컬 저장소를 구성하고 Jenkins와 연동한다.

  • Maven의 설치 및 사용법 ‘Maven’ 자료 참고
  • Git의 설치 및 사용법 ‘Git’ 자료 참고
    • Git 저장소 구성
    • 사용자 등록과 권한을 설정
    • Jenkins에서 형상관리 도구인 Git을 연동하기 위한 설정
    • Jenkins 신규 아이템 추가 및 설정
    • Jenkins에서 빌드 실행

Git bash에서 git clone https://github.com/hanmomhanda/SimpleJavaWebApp.git 를 실행해서 GitHub에 있는 저장소를 D:\swbank\gitRepo 폴더에 복제한다.

Enable security를 체크한다.

Security Realm에서 Jenkins own user database를 선택해서 Jenkins 내부 DB를 사용하도록 하고, 사용자 가입 허용을 선택한다.

Authorization에서 Logged-in users can do anything을 선택하고 Save를 선택한다.

Project-based Matrix Authorization Strategy를 선택하면, 다음과 같이 세부적으로 권한을 관리할 수 있다.

로그인 화면에서 ‘계정 생성’을 선택한다.

사용자 정보 입력 후 Sign Up을 선택한다.

신규 아이템 생성

실행 초기 화면에서 ‘새로운 Item’을 선택한다.

Item 이름을 입력한다.

프로젝트 종류에 따른 설명은 Jenkins 화면에 한글로 잘 나와 있다.

본 자료에서는 Freestyle project를 선택해서 진행한다.

신규 아이템 생성 화면

‘소스 코드 관리’ 항목에서 Git을 선택하고 아래와 같이 입력한다.

  • Repository URL : Git 저장소 경로 (예제에서는 file://D:/swbank/gitRepo/SimpleJavaWebApp)
  • Branch Specifier : 브랜치 이름(예제에서는 master)

‘빌드 유발’ 항목은 그대로 둔다.

  • ‘빌드 유발’ 항목에서 아무 선택도 하지 않으면 Jenkins 웹 화면에서 명시적으로 Build를 실행할 때만 빌드가 수행된다.

옵션

  • 빌드를 원격으로 유발 : 특정 URL에 인증 토큰과 함께 접속하면 빌드 실행
  • Build after other projects are built : 다른 프로젝트의 빌드가 끝나면 이 항목 빌드
  • Build periodically : 주기적으로 빌드 실행
  • Poll SCM : 주기적으로 SCM을 확인해서 달라진 것이 있으면 빌드 실행
  • 주기 옵션 . 분 시 일 월 요일 의 5가지 항목으로 지정.

    예) 매 15분 마다 : H/15 * * * * 월~금요일까지 9~16시 사이에 매 2시간 마다 : H 9-16/2 * * 1-5

‘Build’ 항목의 Add build step을 선택해서 Execute Windows batch command를 추가하고 아래와 같이 입력한다.

  • mvn을 실행하기 위한 JAVA_HOME설정 및 goal을 지정한다.

‘Build’ 항목의 Add build step을 선택하여 Execute Windows batch command를 하나 더 추가하고 아래와 같이 입력한다.

  • 빌드 결과 생성된 war 파일을 웹 애플리케이션 서버에 복사한다.
