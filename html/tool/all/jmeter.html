도구 소개

도구명 Jmeter
(http://jmeter.apache.org/) 		라이센스 APL2
소개
  • 부하 테스트 및 성능 측정을 위해서 개발된 100% 순수 자바 애플리케이션이다.
  • HTTP를 사용하여 사이트의 부하를 테스트 할 수 있으므로 정적이거나 동적인 자원 (파일, 서블릿, 자바객체들, 데이터베이스 등)에 대해서 테스트하는데 사용된다.
특징
  • HTTP, FTP, SMTP, POP3, LDAP, JMS, SOAP, TCP 등 다양한 프로토콜 지원
  • 완벽한 Multi Threading 프레임 워크 지원
  • 다수의 Thread가 동시에 테스트 데이터를 추출
주요 기능
  • 정적, 동적인 웹 페이지 부하 및 성능 테스트
  • Response Assertion
  • Duration Assertion
  • Size Assertion
실행 환경 Windows, Linux, MacOS 사전 설치 도구 Java 8 이상
카테 고리 웹 부하 테스트 최신 버전 v5.0 (2018.09)
관련 도구
  • Grinder, nGrinder, Gatling, TSung

주요 기능

  • HTTP, FTP, SMTP, POP3, LDAP, JMS, SOAP, TCP 등 다양한 프로토콜로 원격의 목적지 서버 및 웹 페이지를 대상으로 부하 테스트 수행
  • 멀티쓰레드 기반 샘플링
  • 테스트 녹화 및 테스트 결과 시각화

설치 환경

Windows의 경우 아래와 같이 Command창에서 ‘java –version’을 실행하여 설치된 JDK 설치 여부를 확인한다.

Java 8.0 이상을 권장한다.

다음의 사이트에서 JMeter를 다운 받는다.

Binary 부분에서 Windows용 JMeter를 다운받는다.

다운받은 파일 압축을 해제한다.

jmeter.bat 파일을 선택하면 다음과 같이 JMeter의 메인 화면이 보인다.

예제 소개

JMeter의 기능을 확인하기 위해 아래 예제를 기반으로 기능을 확인한다.

간단한 HTML 페이지를 사용하여 아래의 기능을 소개한다.

  • 웹 페이지 접속 가능 여부 확인 테스트 기능
  • 웹 페이지 부하 테스트 기능
  • 응답 결과를 검증하기 위한 Assertion 기능

테스트를 위해 Apache 웹 서버를 구축한 후 테스트용 웹 페이지를 Apache 서버에 올려 테스트를 진행한다.

테스트 웹 페이지는 Restaurant에 대한 정보를 입력하는 페이지이며 웹 페이지는 다음과 같이 구성된다.

  • Restaurant Name
  • Restaurant Type
  • Location
  • Credit Cards 사용가능 유/무

테스트용 웹 페이지의 HTML 코드이다.

  • 파일명은 form.html

웹 페이지 접속 가능 여부 확인 테스트 기능

테스트 플랜을 작성한다.

  • 테스트 플랜은 JMeter가 수행할 일련의 실행 코드의 집합이다.
  • JMeter를 사용한 테스트는 이 실행 코드 집합을 알맞게 조합하는 것으로 표현한다.

Thread Group을 만든다.

  • Thread Group이란 테스트 플랜이 수행할 하나의 Group
  • 왼쪽 Test Plan에서 우 선택 → Add → Threads (Users) → Thread Group 선택

Thread Group 생성 후 오른쪽에 나오는 항목들을 작성한다.

  • Name : Thread Group의 이름을 입력한다.
  • Action to be taken after a sampler error : 테스트 수행 중 에러가 발생될 때의 상황을 설정한다.
  • Thread Properties

Number of Threads : 이 Thread Group에 생성될 Thread의 개수(가상 사용자의 수)를 지정한다.

Ramp-up Period : 한 Thread가 시작한 후 다음 Thread가 시작될 때 까지의 대기 시간을 지정한다.

Loop Count : 각 Thread가 Thread Group에 속한 작업의 반복 횟수를 지정한다.

Scheduler : Thread Group의 시작 및 종료 스케줄을 설정할 것 인지 체크한다.

HTTP Request Defaults를 추가한다.

  • Thread Group에서 마우스 우 선택 → Add → Config Element → Http Request Defaults

Http Request Defaults는 Config Element에 속한 요소로서 관련된 Http Requests가 사용할 기본 정보를 지정한다.

  • Http Request Defaults는 그 자체적으로 어떤 기능도 수행하지 않는다.
  • Http Request Defaults는 관련 영역에 있는 모든 Http Requests 요소에 영향을 준다.

Http Request를 추가한다.

Http Request 요소는 HTTP 요청을 서버에 전송하여 그 결과를 받아오는 부분이다.

  • Thread Group에서 우 선택 → Add → Sampler → HTTP Request 선택

Http Request의 입력 정보이다.

  • 입력하지 않은 정보는 앞에서 작성한 Http Request Defaults에 작성된 내용이 반영된다.

테스트 결과를 출력해 주는 Listener를 추가한다.

  • 테스트 결과를 출력하는 Listener로서 View Results Tree를 사용한다.
  • Thread Group선택 후 마우스 우 선택 → Add → Listener → View Results Tree 선택한다.

테스트를 실행한다.

  • 메뉴 바의 Run → Start

Start를 선택 후 왼쪽 트리에서 View Results Tree를 선택한 다음 오른쪽에서 HTTP Request를 선택하면 테스트 결과가 출력된다.

웹 페이지 부하 테스트 기능

부하 테스트란, 서비스가 작동될 서버에 의도적으로 일정량의 트래픽을 발생시켜 서버가 다운되지 않고 정상적으로 작동되는지를 테스트 하는것을 말한다.

처음에 할 일은 테스트 플랜을 작성하는 것이다.

Thread Group을 생성한다.

Thread Group이란 테스트 플랜이 수행할 하나의 Group을 말한다.

  • 왼쪽 Test Plan에서 우 선택 → Add → Threads (Users) → Thread Group을 선택한다.

오른쪽에 나오는 항목들을 작성한다.

  • 부하 테스트를 위해서 Loop Count 부분에서 10000을 입력한다.

Loop Count는 웹 페이지 접속 횟수를 뜻한다.

본 예제에서는 Loop Count에 10000을 입력하였으므로, 웹 페이지에 10000번 접속한다는 것이다.

Http Request Defaults를 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Config Element → Http Request Defaults

아래의 항목들을 작성한다.

Http Request를 추가한다.

  • Thread Group에서 우 선택 → Add → Sampler → HTTP Request 선택한다.

Http Request의 입력 정보

테스트 결과를 출력해 주는 Listener를 추가한다.

  • 2개의 Listener를 추가한다.

View Results Tree

Graph Results

Response Assertion 테스트 기능

Response Assertion이란 해당 요청의 응답 결과가 지정한 문자열을 포함하고 있는 지의 여부를 검증할 때 사용하는 기능이다.

처음에 할 일은 테스트 플랜을 작성하는 것이다.

Thread Group을 생성한다.

  • Test Plan에서 우 선택 → Add → Threads (Users) → Thread Group을 선택한다.

오른쪽에 나오는 항목들을 작성한다.

  • Loop Count 부분에서 1을 입력한다.

Http Request Defaults를 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Config Element → Http Request Defaults

아래의 항목들을 작성한다.

Http Request를 추가한다.

  • Thread Group에서 우 선택 → Add → Sampler → HTTP Request 선택한다.

Http Request의 정보를 입력한다.

  • Http Request에 작성되지 않는 부분은 자동으로 Http Request Defaults에 작성된 내용으로 대체된다.

Response Assertion을 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Assertions → Response Assertion 수행한다.

Response Assertion 항목들을 입력한다.

검사할 문자열을 입력한다.

  • Add 버튼 선택 → Pattern to Test에 검사할 문자열을 입력한다.
  • 본 자료에서는 테스트 웹 페이지에 Hong Gil Dong이라는 단어가 존재하는지 검사한다.

Assertion 테스트 결과를 보기 위해서 Listener를 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Listener →Assertion Results 선택한다.

마지막으로 해야 할 일은 테스트를 실행하는 것이다.

  • 메뉴 바의 Run → Start

테스트 실행 후 왼쪽 Assertion Results를 선택하면 다음과 같은 화면이 출력 된다.

Duration Assertion 테스트 기능

Duration Assertion이란 Http Request가 지정한 시간 내에 응답 결과를 리턴하는지 여부를 검증한다.

처음에 할 일은 테스트 플랜을 작성하는 것이다.

Thread Group을 생성한다.

  • 왼쪽 Test Plan 선택 후 마우스 우 선택 → Add → Threads(Users) → Thread Group

오른쪽에 나오는 항목들을 작성한다.

  • 테스트를 한 번 수행한다는 의미로 Loop Count 부분에서 1을 입력한다.

오른쪽에 나오는 항목들을 작성한다.

  • 테스트를 한 번 수행한다는 의미로 Loop Count 부분에서 1을 입력한다.

Http Request Defaults를 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Config Element → Http Request Defaults

아래의 항목들을 작성한다.

Http Request를 추가한다.

  • Thread Group에서 우 선택 → Add → Sampler → HTTP Request 선택한다.

Http Request의 정보를 입력한다.

  • Http Request에 작성되지 않는 부분은 자동으로 Http Request Defaults에 작성된 내용으로 대체한다.

Duration Assertion을 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Assertions → Duration Assertion을 수행한다.

Duration Assertion 항목을 입력한다.

Duration Assertion 부분에 원하는 시간을 Milliseconds 단위로 입력한다.

  • 본 매뉴얼에서는 원하는 시간으로 10 milliseconds로 입력한다.

Assertion 테스트 결과를 보기 위해 Listener를 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Listener → Assertion Results 선택한다.

마지막으로 해야 할 일은 테스트를 실행한다.

  • 메뉴 바의 Run → Start

테스트 실행 후 왼쪽 Assertion Results를 선택하면 다음과 같은 화면 출력된다.

Size Assertion 테스트 기능

Size Assertion이란 웹 페이지의 크기가 원래 목표한 웹 페이지의 크기를 준수하고 있는지를 확인하는 기능이다.

처음에 할 일은 테스트 플랜을 작성하는 것이다.

Thread Group을 생성한다.

  • 왼쪽 Test Plan 선택 후 마우스 우 선택 → Add → Threads(Users) → Thread Group

오른쪽에 나오는 항목들을 작성한다.

  • 테스트를 한 번 수행한다는 의미로 Loop Count 부분에서 1을 입력한다.

Http Request Defaults를 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Config Element → Http Request Defaults

아래의 항목들을 작성한다.

Http Request를 추가한다.

  • Thread Group에서 우 선택 → Add → Sampler → HTTP Request 선택한다.

Http Request의 정보를 입력한다.

Size Assertion을 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Assertions → Size Assertion을 수행한다.

Size Assertion 항목을 입력한다.

Size in bytes 부분에 원하는 바이트 단위로 값을 입력한 후 Type of comparison에서 비교 타입을 선택한다.

  • 본 매뉴얼에서는 Size in bytes : 1000을 입력하고 Type of comparison은 “=“을 선택한다.

Assertion 테스트 결과를 보기위해 Listener를 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Listener → Assertion Results 선택한다.

마지막으로 해야 할 일은 테스트를 실행하는 것이다.

  • 메뉴 바의 Run → Start

테스트 실행 후 왼쪽 Assertion Results를 선택하면 다음과 같은 화면이 출력된다.

Critical Section Controller

Logic Controller에 Critical Section Controller 기능

Critical Section Controller 아래에 있는 테스팅 항목들은 병렬 처리 되지 않고 순차적으로 한 개의 인스턴스 단위로 실행된다.

CSS/JQuery Extractor

CSS Selector로 웹 페이지 문서 항목의 추출이 가능하다.

ForEach Controller

Start/End index로 반복 범위 지정이 가능하다.

Result Status Action Handler

테스트 샘플 에러 발생해도 다음 쓰레드를 실행하도록 설정 가능하다.

MongoDB 연결

MongoDB 연결 설정을 통해 MongoDB에 대한 부하 테스트도 가능하다.

Async BackendListener

GraphiteBackendListenerClient를 이용해서 Dashboard가 구현된 Graphite 서버에 테스트 결과 값을 전달하고 Dashboard 형식으로 결과 확인이 가능하다.

예제 소개

웹 페이지를 사용하는 사용자들은 관리자 측에 계속해서 웹 페이지를 개선할 것을 요구한다.

사용자들의 요구사항

  • 웹 페이지 접속에 너무 많은 시간이 걸린다.
  • 몇몇 웹 페이지는 존재하지 않는다.
  • 수강신청 기간에는 웹 페이지가 열리지 않는다.

웹 페이지의 관리자인 홍길동씨는 자신의 관리하는 웹에 대해서 몇 가지의 테스트를 수행하기로 결정한다.

  • 웹 페이지에 대한 부하 테스트 기능
  • 응답 결과를 검증하기 위한 Assertion 기능 중 Duration Assertion 테스트 기능
  • 응답 결과를 검증하기 위한 Assertion 기능 중 Size Assertion 테스트 기능

관리자 홍길동씨의 테스트 계획

  • 부하 테스트 수행 시 반복 횟수는 1000번으로 설정한다.
  • Duration Assertion 테스트 수행 시 지정한 시간은 10Millsecond으로 설정한다.
  • Size Assertion 테스트 수행 시 테스트할 웹 페이지 크기가 2,000Bytes보다 작을 것이라고 설정한다.

홍길동씨는 테스트 결과를 바탕으로 웹 페이지 수정 작업을 할 것이다.

테스트 플랜 작성하기

처음으로 홍길동씨가 해야 할 일은 테스트 플랜을 작성하는 것이다.

Thread Group을 생성한다.

  • Test Plan 선택 후 마우스 우 선택 → Add → Threads (Users) → Thread Group 선택한다.

오른쪽 항목들에 대해 작성한다.

  • Loop Count는 1000으로 설정한다.

Http Request Defaults를 작성하기

Http Request Defaults를 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Config Element → Http Request Defaults

Http Request Defaults를 작성한다.

Http Request를 작성하기

Http Request를 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Sampler → HTTP Request 선택한다.

Http Request Defaults를 작성한다.

Duration Assertion 작성하기 

Duration Assertion 테스트 수행을 위한 Duration Assertion을 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Assertions → Duration Assertion을 선택한다.

Duration Assertion 테스트를 위해 원하는 응답 시간을 10 Milliseconds로 입력한다.

Size Assertion 작성하기

Size Assertion 테스트 수행을 위한 Size Assertion을 추가한다.

  • Thread Group 선택 후 마우스 우 선택 → Add → Assertions → Size Assertion 선택한다.

Size Assertion 테스트를 위해 크기는 2,000 Bytes로 설정하였으며 비교 타입은 >으로 설정한다.

Listener 추가하기

결과를 보기 위해 3개의 Listener를 추가한다.

  • View Results Tree Listener를 추가한다.

Thread Group 선택 후 마우스 우 선택 → Add → Listener → View Results Tree를 선택한다.

아래 화면은 Graph Results Listener를 추가하는 화면이다.

Thread Group 선택 후 마우스 우 선택 → Add → Listener → Graph Results 선택

아래 화면은 Assertion Results Listener를 추가하는 장면이다.

Thread Group 선택 후 마우스 우 선택 → Add → Listener → Assertion Results를 선택한다.

테스트 실행하기

마지막으로 테스트를 실행한다.

  • 메뉴 바의 Run → Start 선택한다.
