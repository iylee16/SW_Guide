도구 소개

도구명 FitNesse (http://fitnesse.org/) 라이센스 CPL / 무료
소개 자동화된 테스팅을 위한 프레임 워크인 Fit (Framework for Integrated Tests)을 기반으로 하여 테스트를 수행하고, Fit의 테이블 구조를 통한 웹상에서 테스트 결과를 관리하는 도구이다.
특징
  • 자동화된 테스팅을 위한 강력한 프레임워크
  • 테이블 기반 웹 리포팅
  • 테스트 실행 및 결과 관리(롤백, 백업 등) 및 보고
    • Regression Test
    • Acceptance Testing
    • Function Testing
주요 기능
  • 컴파일 된 파일 (.Class) 을 바탕으로 테스트 수행
  • 테스트 관리 및 보고
실행 환경 Windows, Linux, MacOS 사전 설치 도구 JDK 1.7 이상
카테 고리 테스트 최신 버전 v20180127 (2018.01.27)
관련 도구
  • Fit Engine
  • SLIM Engine

주요 기능

  • Wiki Page
  • Regression Test
  • Eclipse Plug-in
  • 외부 Client 접근 / 테스트
  • 테스트 결과 웹 보고

설치 환경

다양한 OS를 지원한다.

  • Windows : Windows XP / Windows 7 (32, 64-bit 모두 지원)
  • Linux : 32, 64-bit 지원
  • Mac OS X : 32, 64-bit 지원
  • UNIX : 32, 64-bit 지원

JDK, 자바 기반 도구(IDE 등)이 필요하다.

  • 코드 및 플러그인 형태, 도구에 포함되어 있는 형태로 제공된다.

설치 단계

1. FitNesse 다운

http://www.fitnesse.org/에서 FitNesse를 다운 받는다.

  • “Download FitNesse” 링크를 실행하여 다운로드 페이지로 이동한다.

fitnesse-standalone.jar를 선택하여 원하는 위치에 다운로드 한다.

  • 저장 위치 및 사용 방법에 따라 설치 방법이 달라진다.

2. FitNesse 실행

FitNesse Classpath를 설정한다.

  • 시스템(Ctrl+Break) → 시스템속성 → 고급 탭 환경변수 → CLASSPATH항목 → 설치위치 지정

FitNess 서버를 실행 한다.

  • “Java –jar fitnesse.jar -p 8080”명령어 입력 → 가동상태 확인

3. FitNesse 실행 결과

웹 사이트 접속 : http://localhost:8080/

  • FitNesse 초기 접속화면 확인이 가능하다.

Fit, FitNesse

Fit(Framework for integrated)이란?

  • 자동화 테스팅 프레임워크이다.
  • 테스트들을 보여주고 테스트들의 자동 체킹 결과를 보고하기 위한 테이블 사용
    • 비즈니스 관점에서 테스팅
    • 테이블식 구성을 통해 프로그래밍에 대한 배경지식 없이 테스트를 수행할 수 있도록 한다.
      • 이로 인해 전반적으로 필요한 시스템 개발 편의를 제공한다.
  • 범용으로 사용하기 위한 개방적인 프레임 워크로, 여러 가지 방법으로 보여주기 위한 확장을 제공한다.

FitNess

  • 자동화 테스팅 프레임워크 Fit을 손쉽게 사용할 수 있도록 구현한 것이다.
  • FitNesse의 장점
    • 개발 커뮤니케이션 향상에 용이하다.(개발팀에게 손쉽게 재현 요청 처리 가능)
    • 테스트 케이스 추가 작업이 용이하다.
    • Suite로써 Regression Test가 가능하다.

Fit Table 이점

Communication : 필요한 것에 대하여 토론 및 의사 소통 할 수 있는 구체적인 방법 제공한다.

  • 개발된 소프트웨어의 목적 및 필요성, 가능성에 대한 이해를 돕는다.
  • 현실적인 예제를 기반으로 구체적인 테스트를 함으로써 개발 요구사항에 대한 공감대를 형성할 수 있다.

Agility : 요구 변경 및 피드백을 제공 한다.

  • 자동화된 테스트는 변경되는 모든 항목을 정의하는 것을 도와주며 소프트웨어의 변경에 대해서도 이전에 만족했던 요구 항목이 잘못되지 않도록 한다.
  • Fit 테스트는 가장 최근의 문제와 솔루션의 이해에 따라 가장 유용한 피드백을 제공 한다.

Balance : 초기문제 해소 및 변경에 의한 문제 재발 방지를 위한 균형 유지에 용이하다.

  • Fit 테스트들은 변경을 할 필요가 있으나 균형을 잃은 소프트웨어 제품에도 도입될 수 있다.

FitNesse 메인 페이지

FitNess메인 페이지

  • Edit, Properties, Where Used, Search, Files, Versions, Recent Change, User Guide, Test History

1. Edit

첫 페이지(Wiki)페이지를 편집하고, 새로운 테스트 페이지를 생성한다.

  • ">Test”를 입력 → Save

2. Properties

페이지 설정 화면, 각 테스트의 Type, Action 및 기타 환경을 설정한다.

3. Search

제목별, 내용별, property별 검색이 가능하다.

4. Files

FitNesse 폴더에 대해 파일 탐색 및 업로드/다운로드가 가능하다.

5. Versions

테스트 및 페이지 수정 등에 대해 이전 버전(연/월/일/시) 선택이 가능하다.

6. RecentChanges

FitNesse 내부 변화 로그를 출력한다.

7. User Guide

FitNesse의 매뉴얼 페이지이다.

8. Test History

시간별로 테스트 수행 내역을 테이블을 바탕으로하여 출력한다.

예제 소개

예제 시스템 : 텍스트 출력

  • 간단한 자바 코드와 소켓 통신을 통한 문자를 출력한다.

테스트 페이지 생성

Edit를 눌러 새로운 테스트 페이지를 Wiki 페이지에 추가 한다.

  • Edit 선택 → 테스트명(>EchoTestFixture) 입력 → Save

FIT Table 설계

추가된 Wiki 페이지에서 테스트 페이지로 이동한다.

  • 추가된 EchoTestFixture에서 ?를 선택한다.

FIT Table 생성

  • 엑셀에서 작성하여 복사 → Spreadsheet to FitNesse 선택 → Save

FIT Table 생성 결과

  • 엑셀에서 작성된 테이블이 적용된 모습이다.

Fixture 생성

EchoTestFixture.java 생성한다.

  • 테스트 대상이 될 코드 작성 및 컴파일

클래스 파일 연결

테스트할 클래스 위치를 연결(EchoTestFixture.class) 한다.

  • 클래스 파일이 사용한 라이브러리(!path fitnesse.jar)와 클래스(!path !path C:\FitNesse\test)를 추가 한다.

예제 테스트

Test를 눌러 테스트를 수행한다.

  • 노란 삼각형을 선택하여 정보 확인 및 디버깅이 가능하다.

테스트 케이스를 추가한다.

  • 특수기호, 긴 메시지 등 여러가지 형태로 테스트를 수행한 결과이다.
