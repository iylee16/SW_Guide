도구 소개

도구명 Junit
(http://www.junit.org)		 라이센스 Common Public License
소개 Java 기반 단위 모듈 시험 확인 도구이다.
특징
  • 메소드와 같은 단위 모듈 별 테스팅을 가능케 함으로써 코드 품질을 보장
  • 정확한 단위 테스팅을 가능케 함으로써 통합 테스팅 시 회귀 결함을 감소
  • 다른 모듈에 의존하지 않고, 원하는 모듈만 임의의 순서대로 수행
  • JFeature(요구사항개발도구)와 통합되어 요구사항의 정확한 구현 비율 확인
주요 기능
  • Test case 생성 및 실행
  • 버그 추적
실행 환경 Windows, Linux, MacOS 사전 설치 도구 자바 기반 IDE, JDK
카테 고리 단위 테스트 최신 버전 V5.3.1 (2018.09)
관련 도구
  • CppUnit (https://sourceforge.net/projects/cppunit/)
  • Httpunit (http://www.httpunit.org/)
  • TestLink ( https://sourceforge.net/projects/testlink/files/)

주요 기능

  • 단위 테스트 지원
  • TDD 환경 지원
  • UI Report
  • 실시간 code trace 지원

설치 환경

다양한 OS를 지원

  • Windows : Windows XP / Windows 7 (32, 64-bit 모두 지원)
  • Linux : 32, 64-bit 지원
  • Mac OS X : 32, 64-bit 지원
  • UNIX : 32, 64-bit 지원

JDK, 자바 기반 도구(IDE 등)이 필요하다.

  • 코드 및 플러그인 형태, 도구에 포함되어 있는 형태로 제공한다.

설치 단계

1. Junit 다운

https://github.com/junit-team/junit5/ 에서 JUnit을 다운 받는다.

대부분의 JAVA기반 IDE는 JUnit이 탑재(JUnit 미 탑재, 업데이트, 베타버전을 사용시 주로 다운)

2. Junit 설치

압축을 풀고, junit4.10을 Eclipse가 설치된 폴더 내의 plugins 폴더에 복사한다.

※ 본 매뉴얼에서는 Eclipse기준으로 설명한다.

Continuous Integration이란?

※ Test 수행 단계 기능을 설명하기 위해 계산기 프로젝트를 예로들어 사용한다.

계산기는 다음과 같은 기능을 가지고 있다.

  • sum : 두수의 더하기 연산을 수행한다.
  • subtract : 두수의 빼기 연산을 수행한다.
  • multiply : 두수의 곱하기 연산을 수행한다.
  • divide : 두수의 나누기 연산을 수행한다.
  • ceil : 지정된 숫자를 올림 하는 연산을 수행한다. (ex - 3.2의 ceil연산 : 4)
  • floor : 지정된 숫자보다 작거나 같은 가장 작은 정수를 double형태로 반환하는 연산을 수행한다.
    (ex - 1.2의 floor연산 : 1, 1.7의 floor연산 : 1)
  • abs : 지정된 숫자의 절대값을 계산하고 반환하는 연산을 수행한다.

Test Case 작성

테스트하고자 하는 클래스를 생성 : Calculator

  • 프로젝트선택 → 마우스 우 선택 메뉴의 New → Class → Class이름 설정(Calculator) → Finish

Calculator 클래스의 코드를 작성한다.

  • 계산기의 주요 기능을 구현(sample code)

테스트하고자 하는 클래스에 대해 Test Case를 생성한다.

  • 테스트를 실행할 클래스 선택 → 마우스 우 선택 메뉴의 New-> JUnit Test Case

Source folder, Package, Name, Superclass등을 그대로 두고 NEXT

  • Source folder : Test Case 클래스 파일이 저장되는 폴더를 말한다.
  • Package : Java의 패키지를 의미한다.
  • Name : Test Case 클래스의 이름을 의미(기본값 : 테스트대상클래스명 + Test)한다.
  • Superclass : Test Case 클래스가 상속받을 클래스를 의미(기본값 : junit.framework.TestCase)한다.

Calculator클래스 내의 테스트하고자 하는 메소드 체크 → Finish

테스트하고자 하는 메소드의 내용을 구현한다.

  • 테스트 케이스 메소드를 구현하기 : 오른쪽의 assert 메소드 이용
  • JUnit에서 가장 많이 이용되는 단정(assert) 메소드
    • assertEquals(x, y) : x와y가 같으면 테스트 통과
    • assertFalse(b) : b가 false이면 테스트 통과
    • assertTrue(b) : b가 true이면 테스트 통과
    • assertNull(o) : 객체o가 null이면 테스트 통과
    • assertNotNull(o) : 객체o가 null이 아니면 테스트 통과
    • assertSame(ox, oy) : ox와 oy가 같은 객체를 참조하고 있으면 테스트 통과
    • assertNotSame(ox, oy) : ox와 oy가 같은 객체를 참조하고 있지 않으면 통과
    • assertfail : 테스트를 바로 실패처리

테스트하고자 하는 메소드의 내용을 구현한다.

  • Sum의 테스트 예시
    • Calculator.sum(a,b)를 통해서 두수를 더한 값을 반환한다.
    • 예시 값으로 임의의 값 사용 – a:4, b:5
    • assertTrue를 통해 result가9 인지 확인한다.

테스트하고자 하는 메소드의 내용을 구현한다.

  • 전체 메소드에 대해서 구현한다.

Test Suite 작성

메뉴바의 File → New → Other → JUnit선택 → JUnitTestSuite → Next

  • Test클래스를 한데 묶어서 테스트(본 예시), 특정 메소드만 실행하거나 하지않을 때 사용한다.

테스트할 클래스를 선택 → Finish

Test Suite 작성 결과

테스트 실행 및 결과

JUnit 테스트의 실행 : Test Case선택 → RUN → Run As → JUnit Test

  • JUnit Test 실행화면 : 모든 단위테스트에 성공(녹색막대), 실패(붉은색 막대) 표시한다.

JUnit 테스트 실행 화면

  • 테스트 메소드 옆에 성공한 테스트는 v 표, 실패한 테스트의 목록은 x 표로 표시한다.
  • 상단메뉴
    • Runs : 실행된 테스트의 개수이다.
    • Errors : 예외 상황이 발생한 테스트의 개수이다.
    • Failures: 실패한 테스트의 개수이다.
  • 상단 막대
    • 빨간 막대 : 테스트 실패
    • 초록 막대 : 테스트 성공
  • Failure Trace
    • 실패한 부분의 추적 정보를 보여준다.
    • 실패한 테스트의 메소드 및 위치를 확인
    • 실패메시지 선택 : 소스코드 위치로 이동

도구 기능

테스트 실행 메뉴 소개

예제 소개

예제 시스템 : 교통 신호등 프로그램을 구현한다.

  • 교통 신호등 요구사항 : TDD(Test Driven Development)을 이용한 예제
  • 계산기 요구사항 : 빨강, 노랑, 초록 불을 가지고 있는 교통신호등 프로그램을 개발한다.

신호등의 처음 상태는 빨간 불

신호등의 상태 변화는 빨간 불, 빨간 불과 노란 불 동시에, 초록불, 노란불, 다시 빨간 불 순

  • TDD는 정의된 요구사항을 기반으로 Test Case를 먼저 만들고, 이를 기반으로 구현한다.
  • TDD는 처음부터 프로그램을 만드는 것이 아닌, 오류를 가정하고 오류를 해결해 나간다.
    • 점차적으로 안정 된 프로그램을 만들어 가는 방식이다.

프로젝트 생성

요구사항 : “신호등의 처음 상태는 빨간 불”을 만족하는 코드를 작성한다.

  • File → New → Project → Java Project → Next

Project 만들기 : TrafficLightModel

  • Project name에 TrafficLightModel → Finish

테스트 케이스 생성

패키지 생성 : TrafficLightmodel

  • src 선택 → 오른쪽 버튼 선택 후 Package → Package이름을 Name으로 설정 → Finish

Test Case생성 : TrafficLightmodelTest

  • Trafficlightmodel 패키지선택 → New → JUnit Test Case → Finish

Test Case 클래스에 테스트 할 내용을 추가한다.

  • “신호등의 처음 상태는 빨간 불”이라는 요구사항을 구현하기 위해, 테스트 케이스 중 assertEquals(true, a.getRed())를 통해서 검증한다.
  • assertEquals(false, a.getYellow())와 assertEquals(false, a.getGreen())은 false로 설정함으로써 초기값이 노란색과 녹색이 아님을 검증한다.

클래스 생성 : TrafficLightmodel 

  • TrafficLightmodel클래스가 존재하지 않기 때문에 좌측 화면과 같은 오류가 발생한다.
  • Java 에디터에서 툴팁(전구) 아이콘 선택 → Create class ”TrafficLightmodel”을 선택한다.

TrafficLighmodel 클래스의 내용 작성 : 의도적 오류

  • TestCase 클래스에서 getRed()를 true로 설정하였기 때문에, 요구사항이 정확히 구현되었는지 확인하기 위해서 Test 대상메소드를 false로 설정한다.
  • False로 설정하였기에 이 메소드는 에러를 발생한다.
  • 발생된 에러를 통해 getRed()가 잘못 구현되었음을 알고, 그 결과를 바탕으로 getRed()를 수정한다.

테스트 수행

테스트를 수행

  • 상단 메뉴바의 Run → Run As → JUnit Test

오류 수정

  • JUnit뷰의 하단 Failure Trace부분을 선택한다.
    • 테스트케이스 getRed() 호출 부분에 문제가 발생했다는 것을 알 수 있다.
    • 커서를 접근 메소드 위로 옮겨 F3키를 눌러 구현부분 TrafficLightModel.java로 이동한다.
    • 최초 요구사항인 “신호등의 초기 상태는 빨간 불”를 만족시키기 위해 getRed()의 리턴값이 true이여야 하므로 현재의 잘못된 리턴값인 false를 true로 변경한다.

테스트를 재 실행 : Run → Run As → JUnit Test

  • 실패 메시지 미 발생, 녹색 막대로 이상 없음을 표시한다.
