도구 소개

도구명 Git (http://git-scm.com/) 라이센스 General Public License V2
소개
  • 리누스 토발즈가 만든 분산형 버전 관리 시스템이다.
  • 대부분의 공개SW가 Git을 이용해서 관리되고 있다.
  • GitHub, BitBucket, GitLab 등 웹 기반의 다양한 소스 저장소 서비스의 기반이다.
  • 대부분의 IDE에서 Git을 지원한다.
특징
  • 분산형 버전관리 시스템으로 Git 사용자가 Git 저장소를 보유하고 원격과 동기화
  • 거의 모든 명령을 로컬에서 수행하며, branch의 생성/전환/폐기가 빠르다
  • 파일별 변화를 저장하는 다른 버전관리 시스템과 달리 파일시스템 스냅샷을 관리
주요 기능
  • 소스 및 문서 버전 관리 서버 및 클라이언트 기능
  • add, commit, reset, branch, checkout, merge, rebase 등 로컬에서 작업
  • push, pull, fetch 등 원격 작업
실행 환경 Windows, Linux, MacOS 사전 설치 도구 없음
카테 고리 형상관리 최신 버전 V2.18.0 (2018.06)
관련 도구
  • TortoiseGit (https://tortoisegit.org/)
  • SourceTree (https://www.sourcetreeapp.com/)
  • Git Extensions (https://gitextensions.github.io/)

주요 기능

터미널에서 git 명령을 실행할 수 있는 Git Bash

GUI 환경에서 git 명령을 실행할 수 있는 Git GUI

설치 환경

Git는 별도의 사전 설치 도구가 필요 없다.

Linux에서는 배포판 별 설치 도구를 통해 설치할 수 있으며, Windows와 MacOSX 용 설치 파일을 다운로드해서 설치할 수 있다.

설치 단계

1. 다운로드

Linux에서의 설치 (https://git-scm.com/download/linux)

터미널에서 배포판에 맞는 설치 명령을 실행한다.

Windows에서의 설치 (https://git-scm.com/download/win)

설치 파일을 다운로드 및 실행한다.

설치 마법사 시작 및 라이센스 사용에 동의한다.

설치 위치 지정 및 설치 옵션을 지정한다. (기본값 설치)

시작 프로그램 등록 및 PATH 환경 변수를 설정한다. (기본값 설치)

시작 프로그램 등록 및 PATH 환경 변수를 설정한다. (기본값 설치)

개행 문자를 설정한다.

  • Checkout : Windows-style, commit : Unix-style line endings

실행 터미널을 지정한다.

  • Use MinTTY : MinTTY 터미널 새로 설치
  • Use Windows’ default console window : 윈도우 기본 명령창에서 git 실행

파일 시스템 캐쉬 기능

  • 파일 시스템 데이터를 대량으로 읽어서 메모리에 캐쉬
  • 성능을 대폭 향상 시킬 수 있으나, 선택하지 않고 기본값 그대로 설치한다.

설치 시작 및 설치를 완료한다.

git 설정 파일

git 설정 파일은 3단계로 구성되어 있다.

  • /etc/gitconfig : 시스템 모든 사용자와 모든 git 저장소에 적용된다.

git config --system은 이 파일을 수정

  • ~/.gitconfig 또는 ~/.config/git/config : 특정 사용자에게만 적용된다.

git config --global은 이 파일을 수정

  • git_저장소_디렉토리/.git/config : 특정 git 저장소에만 적용된다.

git config는 이 파일을 수정

설정 파일을 직접 수정 또는 git config 명령으로 설정 가능하다.

구체성이 높은 것이 우선 적용된다.

  • git_저장소_디렉토리/.git/config의 설정 사항이 가장 우선 적용되고
  • /etc/gitconfig의 설정 사항은 우선순위가 가장 낮다.

윈도우에서는 $HOME(보통 C:\Users\$USER)/gitconfig 와 git_저장소_디렉토리/.git/config

이렇게 두 가지의 설정 파일이 존재한다.

버전 관리 대상 제외

.gitignore 파일을 통해 특정 파일이나 디렉토리를 버전 관리 대상에서 제외 가능하다.

아래의 파일에 설정된 모든 항목은 버전 관리 대상에서 제외된다.

  • ~/.config/git/gitignore - git_저장소_디렉토리/.gitignore
  • git_저장소_디렉토리/.git/info/exclude
  • git 설정 파일의 core.excludesFile에 명시된 파일

주로 class 파일 등 컴파일된 파일이나 컴파일된 파일이 모여있는 디렉토리를 버전 관리 대상에서 제외한다.

git clone

git 저장소를 복제해서 새 저장소를 생성한다.

git checkout

다른 브랜치나 tag로 이동한다.

새 브랜치를 생성하고 새 브랜치로 이동한다.

git diff

수정 내용을 비교한다.

git status

현재 git 저장소 상태를 확인한다.

git add

수정 내용을 commit 대상으로 등록한다.

git Commit

git add로 등록되어 있는 수정 내역을 한 묶음으로 해서 로컬 저장소에 반영한다.

git merge

다른 브랜치를 병합한다.

git rebase

현재 브랜치의 분기 기준점(base)을 재설정한다.

분기 기준점으로 재설정된 브랜치(1.11-stable)를 checkout해서, rebase를 실행한 브랜치(rebase-text)를 merge 해야한다.

git fetch

원격 저장소에 있는 브랜치를 로컬 저장소로 가져온다.

git pull

원격 저장소에 있는 브랜치를 로컬 저장소로 가져와서 merge 한다.

git push

로컬 저장소에 있는 브랜치를 원격 저장소에 반영한다.
