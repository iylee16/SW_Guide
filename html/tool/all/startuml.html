도구 소개

도구명 StartUML
(http://staruml.io/) 		라이센스 GPL
소개 UML 모델링도구로 Use Case Diagram, Class Diagram, Sequence Diagram등 다양한 Diagram을 간편한 UI를 이용하여 쉽고 빠르게 생성할 수 있도록 편리한 기능을 제공한다.
특징
  • UML 표준명세에 기반 모델 작성과 UML2.0 표기법을 지원
  • 완전한 UML Profile 개념을 제공하여 플랫폼에 독립적인 모델을 작성할 수 있도록 지원
  • 뛰어난 확장성과 유연성을 제공
  • 심플한 인터페이스로 학습 및 사용이 용이하며 개발 효율을 높임
주요 기능
  • Use Case Diagram • Class Diagram • Collaboration Diagram • Statechart Diagram
  • Activity Diagram • Sequence Diagram • Component Diagram • Deployment Diagram
  • Composite Diagram
실행 환경 Windows, Linux, MacOS 사전 설치 도구
카테 고리 설계 모델링 최신 버전 v3.0.2 (2018.08)
관련 도구
  • UMLet, VioletUMLEditor, Rational Rose, Together

주요 기능

UML 모델링 도구로 다양한 Diagram을 간편한 UI를 이용, 쉽고 빠르게 생성할 수 있도록 편리한 기능을 제공한다.

설치 환경

Windows XP / Windows 7(32-bit 프로그램으로 설치)

독립적인 프로그램 (Eclipse 불필요)

플러그인 아키텍처 제공으로 인한 개발환경 플러그-인으로 사용 가능하다.

  • c++, java, c#, rose 등

넓은 확장성으로 인한 넓은 사용 및 발전 가능성이 있다.

  • Open API, Com-based plug-in Architecture, MFC와 J2EE등 모델 프레임워크를 지원한다.

예제 소개

도구 기능 소개 예시 : 수강신청 시스템 구축

  • 수강신청 시스템 구축 요구사항 (일부)

    이 시스템은 수강 신청 페이지에 접속하여 수강 신청을 할 수 있도록 도와주는 것이다.
    시스템에 등록된 학교 학생이 수강 신청 페이지에 로그인 후 과목을 선택하고, 수업을 선택하여 잔여 좌석을 확인한다.
    만약, 잔여 좌석이 없을 경우 과목 선택부터 다시하고 잔여 좌석이 있을 경우 수강신청을 완료하게 된다.

  • 수강신청 시스템의 구현 환경
    • 운영체제환경
      • 사용자는 PC 기반의 Windows XP를 운영체제로 사용한다.
      • Web Server, DB Server, Web Application Server는 Windows NT 기반의 운영체제를 사용한다.
    • 네트워크 환경
      • 사용자와 Web Server간의 통신은 HTTP 방식을 사용한다.
      • 사용자와 Web Application Server간의 통신은 TCP/IP 방식을 사용한다.
      • Web Server와 DB Server, DB Server와 Web Application Server간의 통신은 TCP/IP 방식을 사용한다.

StarUML 사용하기

StarUML을 실행하고 새Project를 생성 : Test

  • Approach는‘Default Approach’를 선택한다.

‘Default Approach’ 실행 결과

  • 기본적으로 5개의 모델에 대하여 UML 생성이 가능하다.

Diagram 생성

  • Model Explorer에서 각Diagram에 적합한 Model선정 → Add Diagram → Diagram선택 → 이름변경을 선택한다.

선택한 Diagram이 Model Explorer에 생성된다.

Tool Box와 Main Window가 Diagram의 종류에 맞게 생성된다.

Use Case Diagram 그리기

수강신청 시스템의 요구사항의 Use Case Diagram 표현한다.

  • 대기 상태에서 수강 신청을 하기위해 수강신청 페이지에 접속한다.
  • 로그인을 하고 과목 목록을 확인한 뒤 과목을 선택한다.
  • 선택한 과목의 수업을 선택하면 수강 신청이 완료된다.

수강신청 시스템의 요구사항의 Use Case Diagram 표현

Use Case Diagram 그리기 : Diagram 생성

  • Model Explorer <<usecaseModel>> Use Case Model 선택 → Add Diagram -> Use Case Diagram을 선택한다.

Actor를 식별

  • Toolbox의 UseCase에서 Actor아이콘 선택 → 마우스로 크기지정 → Actor 이름을 입력한다.

Use Case를 식별

  • Toolbox의 UseCase에서 UseCase아이콘 선택 → 마우스로 크기지정 → Use Case 이름을 입력한다.

Use case 추가 작성

  • 수강신청하기, 수강신청 취소하기, 수강인원 확인하기, 로그인 Use case를 추가한다.

연관관계 표시

  • Toolbox의 UseCase에서 Association 선택 → Actor와 UseCase 마우스 연결한다.

Include관계를 표시

  • Toolbox의 UseCase에서 Include 선택 → Use Case와Use Case 마우스로 연결한다.

Class Diagram 그리기

수강신청시스템의 요구사항의 Class Diagram 표현하기

  • 학교는 0명 이상의 학생으로 구성된 집합연관, 학생은 여러 수업을, 수업은 여러 명의 학생을 수용할 수 있는 다대다 관계 교수 한 명당 0개 이상의 수업을 담당한다.

Class Diagram Tool Bar

Use Case Diagram 그리기 : Diagram 생성

  • Model Explorer의<<designModel>> Design Model 선택 → Add Diagram -> Class Diagram을 선택한다.

시스템을 구성하는 학교클래스 표현

  • Toolbox의 Class에서 Class아이콘 → 마우스로 Class 크기지정 → Class 이름을 입력한다.

학교Class에 Attributes입력

  • 학교Class 선택 → 마우스 오른쪽 버튼 → Collection Editor… → Attributes 탭 → Insert 아이콘 →
    Edit 선택 → 마우스오른쪽버튼 → Name 변경한다.

학교Class에 Operations입력

학교Class 선택 → 마우스오른쪽버튼 → Collection Editor… → Operations 탭 → Insert → Edit → 마우스오른쪽 버튼 → Name 변경한다.

학교 Class 생성 방법을 참고로 교수, 수업, 학생Class를 생성한다.

클래스간의 관계를 표현 : 집합 연관관계(한 클래스가 다른 클래스를 포함하는 관계)

  • Toolbox의 Class에서 Aggregation → Class와 Class 마우스로 연결(드래그)한다.

클래스간의 관계를 표현 : 연관관계(한 클래스와 다른 클래스가 연관 관계가 있음)

  • Toolbox의 Class에서 Association → Class와 Class 마우스로 연결(드래그)한다.

Sequence Diagram 그리기

수강신청시스템의 요구사항의 Sequence Diagram 표현

  • 대기상태에서 수강신청을 하기 위해 수강신청 페이지에 접속한다.
  • 로그인을 하고 과목 목록을 확인한 뒤 과목을 선택한다.
  • 선택한 과목의 수업을 선택하면 수강신청이 완료된다.

Sequence Diagram Tool Bar

Sequence Diagram 그리기 : Diagram생성

  • Model Explorer의 <<analysisModel>> Analysis Model → 마우스오른쪽버튼 → Add Diagram → Sequence Diagram을 선택한다.

상호작용에 참여하는 개체들을 표시

  • Toolbox의 Sequence에서 Object 아이콘 → 마우스로 크기지정 → 이름을 입력한다.

상호 작용에 참여하는 개체들을 표시

  • Student Object 생성하는 방법을 참고하여 Login, application, application management 개체를 표시한다.

다른 객체로의 제어흐름의 이동과 호출 메소드를 표시

  • Toolbox의 Sequence에서 Stimulus아이콘 → Object와 Object선택 → Stimulus이름울 입력한다.

다른 객체로의 제어흐름의 이동과 호출 메소드를 표시

  • Toolbox의 Sequence에서 Stimulus아이콘 → Object와 Object선택 → Stimulus이름을 입력한다.

Activity Diagram 그리기

수강신청 시스템의 요구사항의 Activity Diagram 표현

  • 수강신청 페이지에 접속하여 과목을 선택 한 뒤 수업을 선택하여 잔여 좌석을 확인한다.
  • 잔여 좌석이 없을경우 과목 선택부터 다시 시작한다.
  • 잔여 좌석이 있을경우 수업 등록을 하고 완료한다.

Activity Diagram Tool Bar

Activity Diagram 그리기 : Diagram생성

  • Model Explorer의 각 Diagram에 적합한 Model선정 → 오른쪽버튼 → Add Diagram → Activity Diagram를 선택한다.

객체의 상태 변화가 시작됨을 알린다.

  • Toolbox의 Activity에서 InitialState아이콘 → 마우스로 크기를 조절한다.

작업을 알림

  • Toolbox의 Activity에서 ActionState아이콘 → 마우스로 크기지정 → ActionState 내용을 입력한다.

하나의 상태에서 다른 상태로 변화되는 것을 표현

  • Toolbox의 Activity에서 Transition아이콘 → 위에서 아래로 드래그한다.

ActionState과 Transition 생성 방법을 참고하여 작성한다.

논리식의 결과값에 따라 분기가 일어나는 곳을 표현

  • Toolbox의 Activity에서 Decision선택 → 마우스로 크기를 설정한다.
  • Toolbox의 Activity에서 Transition선택 → 분기가 일어나는 곳과 Decision 부분을 연결한다. (EX : 과목선택)
  • 이어준 선을 선택하여 내용을 입력한다.

객체의 상태 변화가 끝나게됨을 알림

  • Toolbox의 Activity에서 FinalState아이콘 → 마우스로 크기를 지정한다.
  • Toolbox의 Activity에서 Transition아이콘 → 위에서 아래로 드래그한다.

Statechart Diagram 그리기

수강신청 시스템의 요구사항의 Statechart Diagram 표현은 다음과 같다.

  • 대기 상태에서 수강 신청을 하기 위해 수강신청 페이지에 접속한다.
  • 로그인을 하고 과목 목록을 확인한 뒤 과목을 선택한다.
  • 선택한 과목의 수업을 선택하면 수강 신청이 완료된다.

Statechart Diagram Tool Bar

Statechart Diagram그리기 : Diagram 생성

  • Model Explorer의 각 Diagram에 적합한 Model 선정 → 오른쪽버튼 → Add Diagram → Statechart Diagram을 선택한다.

객체의 상태 변화가 시작됨을 알린다.

  • Toolbox의 Statechart에서 InitialState아이콘 → 마우스로 크기를 지정한다.

객체의 상태를 표현

  • Toolbox의 Statechart에서 State아이콘 → 마우스로 State 크기지정 → State 내용을 입력한다.

State 생성 방법을 참고하여 과목 목록 확인, 수업 목록 확인 객체의 상태를 표현한다.

하나의 상태에서 다른 상태로 변화되는 것을 표현

  • Toolbox의 Statechart에서 Transition아이콘 → 연결할 State와 State를 선택한다.

객체의 상태 변화가 끝나게 됨을 알림

  • Toolbox의 Statechart에서 FinalState아이콘 → 마우스로 크기를 지정한다.
  • Toolbox의 Statechart에서 Transition아이콘 → State와 FinalState 연결한다.

객체의 상태 변화를 표현

  • 화살표 선택 → 더블클릭 → 해당내용입력

Collaboration Diagram 그리기

수강신청 시스템의 요구사항의 Collaboration Diagram표현

  • 수강신청 페이지에 접속하여 과목을 선택 한 뒤 수업을 선택하여 잔여 좌석을 확인한다.
  • 잔여 좌석이 없을경우 과목 선택부터 다시 시작한다.
  • 잔여 좌석이 있을경우 수업 등록을 하고 완료한다.

Collaboration Diagram Tool Bar

Collaboration Diagram그리기 : Diagram생성

  • Model Explorer의 각Diagram에 적합한 Model 선정 → 오른쪽버튼 → Add Diagram → Collaboration Diagram을 선택한다.

객체를 생성

  • Toolbox의 Collaboration에서 Object아이콘 → 마우스로 Object 크기 지정 → Object 내용을 입력한다.

객체 생성하는 방법을 참고하여 과목선택, 수업선택, 수강처리, 수강인원확인, 수강완료 객체를 생성한다.

객체 사이의 링크를 생성

  • Toolbox의 Collaboration에서 Link아이콘 → 연결할 Link와 Link사이를 드래그한다.

Link의 상태 변화를 표현

  • Link 선택 더블클릭 → 해당내용입력

예제 소개

예제 시스템 : 생산관리시스템

  • 생산 관리 요구사항(일부)

    업무 담당자는 시스템에 접속하여 발주, 매입, 매출, 회계, 재고, 기준정보 관리가 가능하다.

    승인자는 발주와 매입에 대한 승인을 하며, 시스템 관리자는 매출, 회계, 재고, 기준정보에

    대한 시스템 관리를 수행한다.

    발주가 등록되면 발주를 위한 준비를하여 재고여부 조사 후 상품을 배송한다.

  • 생산 관리 시스템의 구현 환경
    • 운영체제환경
      • 사용자는 PC 기반의 Windows XP를 운영체제로 사용
      • Web Server, DB Server, Web Application Server는 Windows NT 기반의 운영체제를 사용한다.
    • 네트워크 환경
      • 사용자와 Web Server간의 통신은 HTTP 방식을 사용한다.
      • 사용자와 Web Application Server간의 통신은 TCP/IP 방식을 사용한다.
    • Web Server와 DB Server, DB Server와 Web Application Server간의 통신은 TCP/IP 방식을 사용한다.

Use Case Diagram

생산관리시스템의 요구사항을 UseCase로 표현한다.

사용자인 업무 담당자자의 표현 : Actor

  • Tool Bar에서 Actor 선택 → Main Window 영역에서 마우스왼쪽 버튼 → ‘업무담당자’를 입력한다.

발주관리 기능을 UseCase로 표현

  • Tool Bar에서 UseCase 선택 → Main Window 영역에서 마우스 왼쪽 버튼 → ‘발주관리’를 입력한다.

업무담당자 Actor와 발주관리 UseCase의 관계를 표현

  • Tool Bar에서 DirectedAssociation 선택 → Main Window 영역에서 마우스 왼쪽 버튼

매입관리, 매출관리 기능들에 대해서도 앞서 확인한 바와 같이 UseCase를 추가하고, 관계를 표현한다.

승인자와 시스템관리자 Actor를 생성하고, 관련 UseCase와의 관계를 통해 UseCase Diagram을 완성한다.
