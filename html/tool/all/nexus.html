도구 소개

도구명 Nexus
(https://www.sonatype.com/nexus-repository-oss) 		라이센스 EPL License
소개 프로젝트에서 필요로 하는 Repository 구성 도구이다.
특징
  • 직관적이고 뛰어난 사용성의 UI를 제공
  • Artifact들에 대한 Index 및 search 기능을 제공
주요 기능
  • 다수의 외부 Repository 통합 관리
  • m2eclipse 연동 기능
  • 웹 UI를 통한 artifact 등록 및 관리 기능
실행 환경 Windows, Linux, MacOS 사전 설치 도구 JRE 8
카테 고리 형상관리 최신 버전 v3.14.0 (2018.06.21)
관련 도구
  • Maven (http://maven.apache.org/)
  • Ant (http://ant.apache.org/))
  • Gradle (https://gradle.org/)

주요 기능

  • 외부 Repository를 proxy 개념을 통한 단일 repository 연결 포인트로 제공
  • m2eclipse 연동 기능 제공
  • UI 기반 artifact 등록 및 관리 기능

설치 환경

  • windows, Linux, MacOS 별로 별도의 설치 파일 다운 및 설치
  • 사전 JRE 8이상의 도구 설치 필요 ( https://www.oracle.com/technetwork/java/javase/downloads/index.html)

설치 단계

1. Nexus 다운

다운로드 사이트 :https://www.sonatype.com/nexus-repository-oss

2. 프로그램 실행

  • 다운받은 Nexus 파일 압축 해제한다.
  • command 창 위치 이동
    • 시작 → 실행 → “cmd”입력 → Nexus가 설치된 디렉토리로 이동 (nexus 설치 경로\nexus-3.xx\bin)
  • Nexus를 실행한다.
    • nexus.exe /run
  • 접속한다.
    • http://IPaddress:8081/

로그인

  • 인터넷 브라우저를 통해 설치한 nexus 접속 후 우측 상단 Log In 설정한다.
    • 초기 username 및 Password : admin// admin123

사용자 생성 및 권한 부여

  • Admin 접속 후 사용자 추가 및 password 변경이 가능하다.
    • Security→ Users 선택한다.

Repository 추가

  • Nexus는 다수의 Central repository들을 관리할 수 있으며, Proxy 개념을 통해 보다 쉬운 repository 연동 편의성 제공한다.
    • 메뉴 → add 선택 → Proxy repository 선택한다.
  • 추가할 repository 설정값을 입력한다.

    예) Maven의 대표적인 repository 중 하나인 ibibilio 등록한다.

Artifact 등록

  • 프로젝트에서 사용하는 3rd party 라이프러리들 또는 프로젝트에서 생성한 war, jar 등의 패키징 된 maven artifact들을 nexus repository에 등록한다.
    • Upload하기위한 repository를 선택한다.
      • 3rd party repository 선택 후 오른쪽 마우스 메뉴의 Upload Artifact 메뉴를 선택한다.
  • Upload 하고자 하는 artifact 파일 및 pom.xml 파일을 설정한다.
    • Browser 버튼 클릭을 통해 개발자 PC에 위치한 upload할 파일을 탐색하여 지정이 가능하다.
  • Repository 브라우징을 통한 새로 등록된 artifact 확인한다.

Dependency 설정을 위한 Artifact의 pom.xml을 확인한다.

  • 3rd party 또는 프로젝트에서 생성한 artifact를 프로젝트에서 설정하기 위해서 pom.xml에 dependency 정보를 설정해 주어야 한다.

    ※ 설정할 artifact dependency 정보는 nexus 브라우징을 통해 확인할 수 있다.

  • 오른쪽 마우스 클릭 후 download 메뉴 선택을 통해 dependency 정보를 확인 할 수 있다.

    ※ pom.xml 파일의 정보를 복사/붙어넣기를 통해 프로젝트의 pom.xml 파일에 선언하여 해당 라이브러리 사용한다.
