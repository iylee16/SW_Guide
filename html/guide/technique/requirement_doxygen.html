Doxygen 소개

1. Doxygen 소개

C , C++ , JAVA 소스코드의 구조 및 사양을 문서화 해 주는 오픈소스 개발지원 도구이다. 효과적인 문서화를 위해서는 소스코드내에 표준화된 주석을 사용함으로써, 손쉽게 HTML이나 워드 문서가 생성된다. 문서 작성의 공수를 줄이고, 코드와 문서를 통합함으로써 보다 실질적이고 효과적인 Documentation이 가능하도록 해 준다.

2. 개념적 흐름

3. 관련 프로젝트 산출물

Doxygen의 모바일 방법론의 어플리케이션 설계서 작성시 사용될 수 있으며, 특히 어플리케이션 Class 설계 내용을 문서화하는데 사 용된다. Class 설계 내용을 수작업으로 작성할 경우, Doxygen을 사용하지 않아도 무방하다.

설치 (Installation)

1. 셋업 파일 다운로드

Doxygen의 경우 http://www.doxygen.org 또는 http://www.stack.nl/~dimitri/doxygen/download.html 에서 다운로드 받을 수 있다. Doxygen의 경우, Graphviz와 같은 외부 패키지를 이용할 경우, 보다 시각화된 결과를 얻을 수 있어, 설치가 권고된다. Graphviz는 http://www.graphviz.org/Download..php 에서 다운로드 받을 수 있다.

2. 설치

Doxygen 및 Graphviz 모두 설치에 특이할 점은 없으므로, 일반적인 방식으로 설치하면 된다.

단, Graphviz의 경우, 일부 환경에서 설치가 잘 이루어지지 않는 버그가 있으므로, 설치완료 화면이 제대로 나타나지 않는 경우, 설치 시 사용자를 Just me로 하지 말고, Everyone으로 하면 된다.

사용방법

1. 실행방법

시작메뉴 --> 프로그램 --> doxygen --> doxywizard 실행 (본 기법서는 doxygen 버전 : 1.8.3.1을 기준으로 작성되었음) 다음과 같이 실행된 Doxygen 프로그램을 사용하면 된다.

  • Step1 : Doxygen 을 실행시킬 작업폴더를 선택한다.
  • Step2. 실행을 위한 주요 설정 값을 입력한 후 실행한다. (다음 Page부터 설정 방법 설명)
  1. 설정 및 실행 탭
    • Wizard : 마법사(따라하기) 방식으로 쉽게 따 라하기 방식으로 소스코드의 프로젝트 파일 에 대한 정보를 입력한다. 기본적으로 Wizard 탭을 사용하여 문서를 생성한다. Wizard 탭에 대한 상세한 내용은 3.2절을 참 조한다.
    • Expert : Wizard 탭을 사용하여 설정할 수 없 는 상세한 Doxygen 문서화 옵션을 설정할 수 있다. Expert탭은 기본적으로 사용하지 않 는다. 상세 내용은 3.3절을 참조한다.
    • Run:설정이끝나면,이탭을클릭하여문서 생성 기능을 실행시킬 수 있다. 상세 내용은 3.4절을 참조한다.
  2. 상세 내용
    • Wizard 탭의 내용은 크게 프로젝트/모드/출 력/다이어그램 등에 대한 4가지 Topic내용으 로 구분되며, 상세내용은 3.2절을 참조한다.
  3. Topic별로 상세 조건을 설정하는 영역이다.

2. Wizard 설정

1.1 Wizard > Project 설정

입력사항 설명 입력할 것
Project name 프로젝트 이름 Mobile Device Management
Project version or id 프로젝트 버전 1.0
Source code Directory 소스 파일 폴더 C:/SourceCodeLocation
Destination Directory 문서를 출력할 폴더 C:/DoxygenOutput

1.2. Wizard>Mode 설정

Android 어플리케이션 개발의 경우, java 개발환경이므로 “Optimize for Java or C# output”을 선택한다.

1.3. Wizard>Output 설정

기본적으로 HTML을 선택하고, MS-Word 파일의 결과물을 원할 경우에는 “Rich Text Format(RTF)”도 선택한다.

1.4. Wizard>Diagram 설정

Diagram 생성을 위한 설정을 하는 topic 화면이다. 일반적인 Built-in class diagram generator를 사용해도 무방하지만, Graphiviz Package를 이용할 경우, 미려한 다이어그램 생성이 가능한다.

3. Expert 설정

Graphviz와 같은 외부 패키지를 이용할 경우, 때에 따라 문서생성이 실패하는 경우도 있다. 대부분 긴 이름을 지원하지 못해서 발생 하는 경우이며, 이 경우, Expert > Project > Short_Names의 체크를 활성화시켜 주면 해결된다.

4. Run 실행

Doxygen을 사용하여 문서를 생성한다.

주석적용법

1. 주석 적용 목적

Doxygen을 사용하여 소스코드의 구조 및 사양을 효과적으로 문서화하기 위해서는, 기본적인 주석 표준을 반드시 사용하여야 한다. 이런 주석표준이 없이는 소스코드 내부의 구조적인 내용만을 Doxygen이 문서화하고, 사양이나 설명(Description)이 없게 되므로, 반 드시 주석에 대한 표준을 아래와 같이 적용하여 소스코드를 작성하여야 한다. 아래 내용은 모바일 방법론에서 Doxygen을 사용하여 어플리케이션 설계 내용을 문서화할 경우, 필수 적용사항이며, 수작업으로 작성할 경우 재정의(Tailoring)의 대상이 된다.

2. 주석 적용 기본 형식

일반적인 주석 형식이 /* ~~~ */ 인데 반해, Doxygen 주석 표준은 /** ~~~ */ 로 시작시 *가 하나 더 붙는다.
