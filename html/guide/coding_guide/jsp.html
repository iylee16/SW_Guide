개요

프로젝트가 점점 복잡해지고 규모가 커지면서 개발자가 혼자 코드를 작성하기 보다는 여러 개발자들이 함께 작업하면서 서로 협업하여 시스템을 만들어 나가는 것이 일반화 되었다.

이러한 상호간의 공동작업이 중요해지면서 자신의 코드뿐만 아니라 타 개발자의 코드도 함께 분석하고 리뷰하여 문제점을 함께 파악하고 공동으로 해결해 나가는 경우가 많아지고 있다.

이러한 개발환경에서 코드 스타일을 통일하는 것은 소스코드의 가독성을 높이고 오류의 가능성을 줄이는 중요한 역할을 하게 된다.

본 문서는 Sun의 코딩 표준 권고안 및 기타 권고안을 참조로 하여 작성되었다.

이 문서에서 기술된 내용은 POSCOICT 각 사업부에서 정보시스템 구현 시 개발자들이 숙독하고 코딩 시 적용할 것을 권고한다.

단 포스코, 전자정부 프로젝트 등 고객사에 표준으로 가지고 있는 코딩 가이드가 존재하고 고객사가 자사 표준 코딩 가이드 준수할 것을 요구할 시에는 해당 고객사 코딩 가이드 적용을 우선 원칙으로 한다.

프로그램 Layout

1. 소스 파일 Layout

JSP와 HTML Page는 다음과 같은 Layout을 갖는다

  1. 파일 헤더
  2. Page Directive (<%@ … %>)
    • A. Page Content Type
    • B. Import(한 Line에 하나의 Import만 기술)
  3. Tag Library URI
  4. Page Contents (< HTML > … </HTML>)

2. 주석

  1. Scriptlet(<% … %>) 내의 주석

    Scriptlet내부에는 Java code가 위치하기 때문에, Java Code에서 정의한 주석 Style에 따른다.

  2. Scriptlet(<% … %>) 외의 주석

    HTML 구성요소들 사이에 위치한 주석은 JSP 주석 Style <%-- … --%> 을 따른다. 주석 내용이 단일 행으로 표현 가능한 경우는, <%-- … --%>등 모든 주석을 한 행에 표현하며, 2 행 이상의 주석이 필요한 경우는, <%-- … --%>을 각각 독립된 행에 표현하고, 내용은 별도의 행에 표현한다.

단일 행 주석 예시

<%-- 단일 행 주석 --%>

여러 행 주석

<%--
주석 내용 1
주석 내용 2 …
--%>

기본 Coding

  1. 수식연산(<%= … %>)을 제외한 모든 Scriptlet은 반드시 1 컬럼에서 시작하고, 1 컬럼에서 닫는다. 단, Java Code가 1 Line인 경우는, 동일 Line에서 닫는 것을 허용한다.
  2. 모든 page directive(<%@ page …)는 1 컬럼에 기록한다.
  3. Import는 한 line에 하나만 기술한다.
