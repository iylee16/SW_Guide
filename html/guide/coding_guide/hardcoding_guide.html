개요

UI 컴포넌트 적용 기술

  • HardCoding이란 데이터를 코드 내부에 직접 입력하는 것. 기술적으로는 데이터가 실행 바이너리(exe 파일 등)에 합쳐져 있는 상태를 지칭합니다. 프로그램의 소스 코드에 데이터를 직접 입력해서 저장한 경우, 즉 모든 '상수'는 하드코딩이라고 할 수 있습니다. '변수'의 초기값이나 기본값도 하드 코딩의 범주에 포함됩니다
  • 보안적인 측면에서의 HardCoding의 문제점과 가이드는 Secure Coding Guide 파트를 참조하면 되며, 보안적인 이슈외에도 프로그램의 유연성 등 문제가 있을 수 있는 사항과 예외 적용에 대해서는 본 가이드를 참조하시면 됩니다

Hard Coding 가이드

1. 하드코딩가이드 1

소스유형 HardCoding 유형 점검 대상 설명 비고
UI, SSS 사용자 Message Hard Coding 화면 Message(Java script 등) 사용자 Message : alert, confirm, tooltip
  • alert('비밀번호가 일치하지 않습니다.');
  • password = prompt('System 사용중 Layout No와 SCC의 사용중 Layout No가 다를 경우만 사용하는 기능입니다. 비밀번호를 입력해주세요.')
  • objOption.text = 'Dummy 이적';
UI, SSS 비지니스코드 Hard Coding 항목코드 Value 정의 표준항목의 Value를 정의하기 위해 처리한 유형
  • 사소구분 : public static FACTORY_CODE_PK[] = {'A', 'B','C'};
  • 판매실적 Item 코드 : SaleItemCd = {'SS'};
  • 지시 Error원인 코드 : ErrCauCd = {'HH'};
SSS 비지니스코드 Hard Coding 항목코드 Value 설정 값을 설정하기 위해 처리한 유형
  • 재료상태구분 : setMatTp('A');
  • 외형구분 : this.setShpTp('B');
UI, SSS 비지니스코드 Hard Coding 항목코드 코드 비교 표준항목의 Value를 조건으로 판단 등을 처리한 유형
  • 공장코드 : if(!'A'.equals(facOpCd))
  • 물류코드 : 'B'.equals(logiFlag)
  • 원료구분 : if('19'.equals(geyRawFlag()))
SSS 비지니스코드 Hard Coding 조건 Check용 분기 및 검색조건을 위해 사용한 경우. 아래 예의 경우, 'ABC'가 HardCoding임
  • whereBuffer.append('FAC_CD = "ABC"');

2. 하드코딩가이드 2

소스유형 HardCoding 유형 점검 대상 설명 비고
UI, SSS 비지니스코드 Hard Coding 공통코드(Master Code) 미사용 Case 공통코드(Master Code)가 있는데 사용하지 않은 경우
  • if('B7'.equals(deliveryLotStatus))
UI, SSS 비지니스코드 Hard Coding 공통코드(Master Code) 미적용 비지니스코드를 공통/기준코드화(Master화)를 하지 않고 사용한 경우
  • polCdNameMap.put('AA', '소재 두께이상');
UI 비지니스코드 Hard Coding 공통코드(Master Code) 미사용 Case 화면 LOV에 값을 Load할때 코드/코드명을 Hard Coding
  • 테이블의 Data를 Select해서 Load해야 함

주) 1. UI: JSP, ADF, Forms 등 UI Side 소스코드, 2. SSS:JAVA, PL/SQL, Pro-c 등 Server Side Source Code

주) 공통코드(Master Code): 비즈니스코드를 분류화해서 별도의 코드테이블에 관리하고 관련 프로그램에서 함께 사용하는 코드(표준항목 코드)이며 기준코드라고 부르는 경우도 있음.

Hard Coding 예외 사항

1. 하드 코딩 적용 예외 사항 가이드 1

유형 설명 비고
성능 목적 성능상의 이유로 Hard Coding이 불가피한 경우

ex) 결산처리등 성능을 요하는 프로그램릉 Hard Coding 예외 인정

  • Concurrent Job에서 성능목적으로 Message를 Hard Coding 한 경우
ERP Extension의 경우, 고정불변의 Oracle Seed Data ERP Extension의 경우, 고정불변의 Oracle Seed Data를 Hard Coding한 경우

ex1) transaction_id, event_code등

ex2)

  • 입하/입고 : 'REC', 'DEL' 등
  • Order type: 'BLANK', 'STD'
 ERP에만 해당
업무적으로 고정불변의 비즈니스(표준항목) 코드 업무적으로 향후 변경이 되지 않는 고정불변의 비즈니스(표준항목)코드를 HardCoding한 경우
  • 불편함을 증명해야 하므로 되도록 Hard Coding 제거를 권고함
  • 개발팀은 관련 사항에 대한 기술적인 검토 후 적용을 권고함
단순 Flag 단순 Flag인 'Y', 'N'등은 점검 제외
  • Y/N, Yes/No, true/false 등
ERP Extension의 경우, Lookup Table 조회 조건 Lookup Table에 비즈니스 코드를 저장해두고 사용하는 경우, 비즈니스코드를 select하기 위한 조회 조건은 Hard Coding 점검 대상 제외

ex1) LOOKUP_TYPE = 'TYPE_CD'

ex2) where organization_id = common_pkg.get_value_f('ORGANIZATION_TYPE', 'AB', '', 'cd_attribute')

2. 하드 코딩 적용 예외 사항 가이드 2

유형 설명 비고
Select 문장안에 있는 Code Select 문장안에 있는 고정불변의 Code값은 점검제외
  • QA가 판단하기 어려운 경우, 일단 HardCoding으로 분류 후 개발팁과 협의하여 판단
  • where절에 사용되는 Code값은 상수 선언해서 구현하거나 바인딩으로 처리.
상수 정의 파일 내의 HardCoding ConstantsIF.java, Constants.java, Constant.java, CostIF.java, Const.java, IF.java 파일 내에는 HardCoding 체크하지 않음
  • 비즈니스 중복 정의 여부는 체크하며 ConstantsIF.java라는 접미사 사용 권고
기타 시스템 처리를 위한 고정불변값 고정불변값인 Interface ID, TC, ID, 프로그램 순서제어, 프로그램 ID은 점검 제외
Log Message Package(PL/SQL) 내 Log Message
  • 1회성 및 Biz 적으로 변경 요소가 없는 경우 점검대상 제외.
    단, 영문으로 체인안에서 통일된 형식(문구)를 사용 할 것
Developer Log 유지보수 용이성을 위한 개발자용 Log
html tag html tag는 HardCoding아님
변수초기화중 일부 유형 변수 초기화중 아래는 HardCoding점검 제외
  • ex) 변수명 = 0; 변수명 = ''; put(변수명, '');
Java에서 Main Method내의 테스트용 HardCoding 유지보수에 도움이 되며, HardCoding점검 제외
For 문의 Index 변수 For문의 Index변수는 HardCoding이 아님
  • ex) for i = 1 to 100 // i = 1 HardCoding이 아
